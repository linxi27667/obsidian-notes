# LVGL 父级与子级关系

## 核心概念

### 父子层级关系
- **子级显示在父级上面** - 子对象会覆盖父对象的显示区域
- **通过父级可以获取子级** - 使用下标方式遍历获取子对象
- **不推荐用全局变量** - 将 20 个 widget 作为函数参数传递不是理想做法

---

## 一、对象树结构

LVGL 采用树形结构管理对象，每个对象都有唯一的父对象和多个子对象。

```
屏幕 (Screen)
  │
  ├── 容器 (Container)
  │     ├── 按钮1 (Button)
  │     ├── 按钮2 (Button)
  │     └── 标签 (Label)
  │
  └── 图片 (Image)
```

---

## 二、获取父级/屏幕

### 2.1 获取对象的屏幕

```c
lv_obj_t * screen = lv_obj_get_screen(widget);
```

| 参数    | 类型          | 说明     |
| ----- | ----------- | ------ |
| `obj` | `lv_obj_t*` | 目标对象指针 |

**返回值**: `lv_obj_t*` - 对象所属的屏幕（根父对象）

**说明**:
- 屏幕是最顶层的对象，没有父对象
- 任何对象的屏幕可以通过此函数追溯到

---

### 2.2 获取直接父对象

```c
lv_obj_t * parent = lv_obj_get_parent(widget);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t*` | 目标对象指针 |

**返回值**: `lv_obj_t*` - 对象的直接父对象指针

---

## 三、设置父级

### 3.1 移动对象到新父级

```c
lv_obj_set_parent(widget, new_parent);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t*` | 要移动的对象指针 |
| `parent` | `lv_obj_t*` | 新的父对象指针 |

**说明**:
- 对象会移动到新父对象中
- 移动后，对象会成为新父对象的**最顶层子对象**（最后一个子对象）
- 原父对象的子对象列表中会移除该对象

**示例**:
```c
lv_obj_t * old_parent = lv_obj_create(lv_scr_act());
lv_obj_t * new_parent = lv_obj_create(lv_scr_act());
lv_obj_t * child = lv_obj_create(old_parent);

// 将 child 从 old_parent 移动到 new_parent
lv_obj_set_parent(child, new_parent);
```

---

## 四、获取子级

### 4.1 通过索引获取子对象

```c
lv_obj_t * child = lv_obj_get_child(parent, idx);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `parent` | `lv_obj_t*` | 父对象指针 |
| `idx` | `int32_t` | 子对象索引 |

**返回值**: `lv_obj_t*` - 对应索引的子对象指针

**索引说明**:
| 索引值 | 含义 |
|------|------|
| `0` | 第一个创建的子对象（最底层） |
| `1` | 第二个创建的子对象 |
| `-1` | 最后一个创建的子对象（最顶层） |
| `-2` |倒数第二个子对象 |

---

### 4.2 通过类型过滤获取子对象

```c
lv_obj_t * child = lv_obj_get_child_by_type(parent, idx, &lv_slider_class);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `parent` | `lv_obj_t*` | 父对象指针 |
| `idx` | `int32_t` | 在过滤类型中的索引 |
| `class_p` | `const lv_obj_class_t*` | 要过滤的类型类指针 |

**说明**:
- 只返回指定类型的子对象
- 可用类型: `lv_slider_class`, `lv_btn_class`, `lv_label_class` 等

---

### 4.3 遍历所有子对象

```c
uint32_t i;
uint32_t count = lv_obj_get_child_count(parent);

for(i = 0; i < count; i++) {
    lv_obj_t * child = lv_obj_get_child(parent, i);
    /* 对 child 进行操作 */
}
```

| 函数                               | 说明      |
| -------------------------------- | ------- |
| `lv_obj_get_child_count(parent)` | 返回子对象总数 |
|                                  |         |

## 五、查找子对象（命名系统）

### 5.1 通过名称查找

**前提**: 需在 `lv_conf.h` 中启用 `LV_USE_OBJ_NAME`

#### 查找直接子对象
```c
lv_obj_t * child = lv_obj_get_child_by_name(parent, "button");
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `parent` | `lv_obj_t*` | 父对象指针 |
| `name` | `const char*` | 子对象名称 |

**返回值**: `lv_obj_t*` - 匹配名称的子对象指针，未找到返回 `NULL`

#### 通过路径查找嵌套子对象
```c
lv_obj_t * child = lv_obj_get_child_by_name(cont, "buttons/mybtn_2");
```

**说明**:
- 使用 `/` 分隔路径层级
- `"buttons/mybtn_2"` 表示: cont 的子对象 buttons 的子对象 mybtn_2

---

### 5.2 深度查找（任意层级）

```c
lv_obj_t * widget = lv_obj_find_by_name(parent, "mybtn_1");
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `parent` | `lv_obj_t*` | 搜索起始父对象 |
| `name` | `const char*` | 要查找的对象名称 |

**说明**:
- 在 parent 的所有后代中搜索（不限于直接子对象）
- 适用于忽略层级结构、仅按名称搜索的场景

---

## 六、对象命名系统

### 6.1 设置对象名称

```c
lv_obj_set_name(obj, "my_button");
lv_obj_set_name_static(obj, "my_button");  // 静态版本
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t*` | 目标对象指针 |
| `name` | `const char*` | 名称字符串 |

**两种版本区别**:
| 函数 | 说明 |
|------|------|
| `lv_obj_set_name()` | LVGL 会分配内存存储名称副本 |
| `lv_obj_set_name_static()` | 只存储指针，名称字符串必须保持有效 |

---

### 6.2 自动命名规则

| 规则 | 结果 |
|------|------|
| 未设置名称 | 默认为 `<widget_type>_#`，如 `lv_button_0` |
| 名称以 `#` 结尾 | 自动添加索引，如 `mybtn_#` → `mybtn_0`, `mybtn_1` |
| 自定义名称 | 使用指定名称 |

**命名示例**:
```
container "cont"
  │
  ├── container "header"
  │     ├── label (未命名) → "lv_label_0"
  │     ├── label "title" → "title"
  │     └── label (未命名) → "lv_label_1"
  │
  └── container "buttons"
  │     ├── button (未命名) → "lv_button_0"
  │     ├── button "second_button" → "second_button"
  │     ├── button (未命名) → "lv_button_1"
  │     ├── button "lv_button_#" → "lv_button_2"
  │     ├── button "mybtn_#" → "mybtn_0"
```

**注意**: 索引是"流动的"，如果 `mybtn_2` 被删除，原来的 `mybtn_3` 会变为 `mybtn_2`

## 七、对象位置管理

### 7.1 获取对象在父级中的索引

```c
int32_t idx = lv_obj_get_index(widget);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t*` | 目标对象指针 |

**返回值**: `int32_t` - 对象在父级中的索引（等同于"兄长"数量）

---

### 7.2 移动对象到指定索引

```c
lv_obj_move_to_index(widget, idx);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t*` | 要移动的对象指针 |
| `idx` | `int32_t` | 目标索引 |

**索引说明**:
| 索引值 | 含义 |
|------|------|
| `0` | 移到最底层（最老的子对象位置） |
| `-1` | 移到最顶层（最年轻的子对象位置） |

---

### 7.3 交换两个对象位置

```c
lv_obj_swap(widget1, widget2);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj1` | `lv_obj_t*` | 第一个对象指针 |
| `obj2` | `lv_obj_t*` | 第二个对象指针 |

**说明**: 交换两个对象在父级中的显示顺序

---

## 八、完整示例

### 示例1：遍历子对象并修改属性

```c
lv_obj_t * parent = lv_obj_create(lv_scr_act());

// 创建3个子对象
lv_obj_t * child1 = lv_obj_create(parent);
lv_obj_t * child2 = lv_obj_create(parent);
lv_obj_t * child3 = lv_obj_create(parent);

// 遍历并设置所有子对象的大小
uint32_t count = lv_obj_get_child_count(parent);
for(uint32_t i = 0; i < count; i++) {
    lv_obj_t * child = lv_obj_get_child(parent, i);
    lv_obj_set_size(child, 50, 50);
}
```

---

### 示例2：使用命名系统管理对象

```c
// 启用命名系统 (lv_conf.h)
// #define LV_USE_OBJ_NAME 1

lv_obj_t * cont = lv_obj_create(lv_scr_act());
lv_obj_set_name(cont, "main_container");

lv_obj_t * btn1 = lv_btn_create(cont);
lv_obj_set_name(btn1, "ok_button");

lv_obj_t * btn2 = lv_btn_create(cont);
lv_obj_set_name(btn2, "cancel_button");

// 在其他函数中查找按钮
void handle_buttons(void) {
    lv_obj_t * ok_btn = lv_obj_get_child_by_name(cont, "ok_button");
    lv_obj_t * cancel_btn = lv_obj_find_by_name(cont, "cancel_button");
    
    // 对按钮进行操作...
}
```

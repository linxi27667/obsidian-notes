# LVGL 事件系统 （交互响应与消息传递机制）

## 核心概念

- **事件驱动** - LVGL采用事件驱动架构，用户交互和状态变化通过事件通知
- **回调函数** - 通过注册回调函数响应特定事件，实现松耦合的交互逻辑
- **事件冒泡** - 子控件事件可向上传递给父控件，简化事件处理
- **事件渗透** - 父控件事件可向下传递给子控件，实现全局事件响应
- **手动触发** - 支持代码手动发送事件，模拟用户交互或自定义通知

## 一、事件注册与移除

### 1.1 注册事件回调

```c
lv_obj_t * btn = lv_button_create(lv_screen_active());
lv_obj_add_event_cb(btn, my_event_cb, LV_EVENT_CLICKED, user_data);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t*` | 目标控件 |
| `cb` | `lv_event_cb_t` | 回调函数 |
| `filter` | `lv_event_code_t` | 事件过滤码，`LV_EVENT_ALL`接收所有事件 |
| `user_data` | `void*` | 用户数据，可在回调中获取 |

```c
static void my_event_cb(lv_event_t * event)
{
    printf("Clicked\n");
}
```

**注册多个事件**：

```c
lv_obj_add_event_cb(widget, my_event_cb_1, LV_EVENT_CLICKED, NULL);
lv_obj_add_event_cb(widget, my_event_cb_2, LV_EVENT_PRESSED, NULL);
lv_obj_add_event_cb(widget, my_event_cb_3, LV_EVENT_ALL, NULL);  // 接收所有事件
```

**同一回调不同数据**：

```c
lv_obj_add_event_cb(widget, increment_on_click, LV_EVENT_CLICKED, &num1);
lv_obj_add_event_cb(widget, increment_on_click, LV_EVENT_CLICKED, &num2);
```

> **注意**：事件按注册顺序调用。同一回调可注册多次，每次可携带不同 `user_data`。

### 1.2 注册到显示设备和输入设备

```c
lv_display_add_event_cb(disp, event_cb, LV_EVENT_RESOLUTION_CHANGED, NULL);
lv_indev_add_event_cb(indev, event_cb, LV_EVENT_CLICKED, NULL);
```

### 1.3 移除事件

```c
uint32_t i;
uint32_t event_cnt = lv_obj_get_event_count(widget);
for(i = 0; i < event_cnt; i++) {
    lv_event_dsc_t * event_dsc = lv_obj_get_event_dsc(widget, i);
    if(lv_event_dsc_get_cb(event_dsc) == some_event_cb) {
        lv_obj_remove_event(widget, i);
        break;
    }
}
```

| 函数 | 说明 |
|------|------|
| `lv_obj_get_event_count()` | 获取事件数量 |
| `lv_obj_get_event_dsc()` | 获取事件描述符 |
| `lv_event_dsc_get_cb()` | 获取回调函数 |
| `lv_obj_remove_event()` | 移除指定事件 |

## 二、事件类型详解

### 2.1 输入设备事件

| 事件码 | 触发时机 | 典型应用 |
|--------|---------|---------|
| `LV_EVENT_PRESSED` | 控件被按下 | 按钮按下反馈 |
| `LV_EVENT_PRESSING` | 持续按下中（连续触发） | 长按连续操作 |
| `LV_EVENT_PRESS_LOST` | 按下但滑出控件 | 取消操作检测 |
| `LV_EVENT_SHORT_CLICKED` | 短按后释放（未滑动） | 单击确认 |
| `LV_EVENT_SINGLE_CLICKED` | 单击序列中的第一次 | 单击计数 |
| `LV_EVENT_DOUBLE_CLICKED` | 双击 | 双击切换 |
| `LV_EVENT_TRIPLE_CLICKED` | 三击 | 三击特殊操作 |
| `LV_EVENT_LONG_PRESSED` | 长按达到阈值 | 长按菜单 |
| `LV_EVENT_LONG_PRESSED_REPEAT` | 长按后周期性触发 | 长按连续增减 |
| `LV_EVENT_CLICKED` | 释放且未滑动（通用点击） | 通用点击处理 |
| `LV_EVENT_RELEASED` | 任何情况下的释放 | 释放清理 |
| `LV_EVENT_SCROLL_BEGIN` | 开始滚动 | 滚动动画 |
| `LV_EVENT_SCROLL_THROW_BEGIN` | 开始惯性滚动 | 惯性处理 |
| `LV_EVENT_SCROLL_END` | 滚动结束 | 滚动完成处理 |
| `LV_EVENT_SCROLL` | 滚动中 | 实时滚动响应 |
| `LV_EVENT_GESTURE` | 检测到手势 | 手势导航 |
| `LV_EVENT_KEY` | 按键输入 | 键盘控制 |
| `LV_EVENT_FOCUSED` | 获得焦点 | 焦点高亮 |
| `LV_EVENT_DEFOCUSED` | 失去焦点 | 焦点移除 |
| `LV_EVENT_LEAVE` | 离开控件但仍被选中 | 离开处理 |
| `LV_EVENT_HOVER_OVER` | 鼠标悬停进入 | 悬停效果 |
| `LV_EVENT_HOVER_LEAVE` | 鼠标悬停离开 | 悬停移除 |

### 2.2 绘制事件

| 事件码 | 触发时机 | 说明 |
|--------|---------|------|
| `LV_EVENT_COVER_CHECK` | 检查控件是否完全覆盖区域 | 参数为 `lv_cover_check_info_t*` |
| `LV_EVENT_REFR_EXT_DRAW_SIZE` | 获取额外绘制区域 | 参数为 `int32_t*`，用于阴影等 |
| `LV_EVENT_DRAW_MAIN_BEGIN` | 主绘制阶段开始 | |
| `LV_EVENT_DRAW_MAIN` | 执行主绘制 | |
| `LV_EVENT_DRAW_MAIN_END` | 主绘制阶段结束 | |
| `LV_EVENT_DRAW_POST_BEGIN` | 后绘制阶段开始（子控件绘制后） | |
| `LV_EVENT_DRAW_POST` | 执行后绘制 | |
| `LV_EVENT_DRAW_POST_END` | 后绘制阶段结束 | |
| `LV_EVENT_DRAW_TASK_ADDED` | 添加绘制任务 | |

> **警告**：在绘制事件回调中，禁止修改控件属性、样式或创建/删除控件，否则会导致断言失败。

### 2.3 特殊事件

| 事件码 | 触发时机 | 典型应用 |
|--------|---------|---------|
| `LV_EVENT_VALUE_CHANGED` | 控件值改变 | 滑块、开关状态变化 |
| `LV_EVENT_INSERT` | 文本插入 | 文本输入处理 |
| `LV_EVENT_REFRESH` | 刷新通知 | 手动刷新控件 |
| `LV_EVENT_READY` | 过程完成 | 操作完成回调 |
| `LV_EVENT_CANCEL` | 过程取消 | 取消操作处理 |
| `LV_EVENT_STATE_CHANGED` | 状态改变 | 状态切换响应 |

> **💡 性能优化提示**：对于滑块等频繁变化的控件，`LV_EVENT_VALUE_CHANGED` 会在每次值变化时触发。如果回调中包含耗时操作（如网络请求、Flash写入），建议使用 `LV_EVENT_RELEASED` 或 `LV_EVENT_CLICKED` 替代，或采用"实时显示+释放执行"的组合策略。详见示例4。

### 2.4 其他事件

| 事件码 | 触发时机 |
|--------|---------|
| `LV_EVENT_CREATE` | 控件创建中 |
| `LV_EVENT_DELETE` | 控件删除中 |

| `LV_EVENT_SIZE_CHANGED` | 控件尺寸改变 |
| `LV_EVENT_STYLE_CHANGED` | 控件样式改变 |
| `LV_EVENT_LAYOUT_CHANGED` | 布局重新计算 |
| `LV_EVENT_GET_SELF_SIZE` | 获取控件内部尺寸 |

### 2.5 显示设备事件

| 事件码 | 触发时机 |
|--------|---------|
| `LV_EVENT_INVALIDATE_AREA` | 区域标记为需重绘 |
| `LV_EVENT_RESOLUTION_CHANGED` | 分辨率改变 |
| `LV_EVENT_COLOR_FORMAT_CHANGED` | 颜色格式改变 |
| `LV_EVENT_REFR_REQUEST` | 请求刷新 |
| `LV_EVENT_REFR_START` | 刷新周期开始 |
| `LV_EVENT_REFR_READY` | 刷新完成（Flush后） |

## 三、事件数据获取

### 3.1 lv_event_t 数据获取函数

```c
static void my_event_cb(lv_event_t * e)
{
    // 获取事件码
    lv_event_code_t code = lv_event_get_code(e);

    // 获取当前目标控件（事件处理者）
    lv_obj_t * current_target = lv_event_get_current_target(e);

    // 获取原始触发目标
    lv_obj_t * target = lv_event_get_target(e);

    // 获取原始触发控件（推荐C++使用）
    lv_obj_t * target_obj = lv_event_get_target_obj(e);

    // 获取用户数据
    void * user_data = lv_event_get_user_data(e);

    // 获取发送事件时传入的参数
    void * param = lv_event_get_param(e);
}
```

| 函数 | 返回值 | 说明 |
|------|--------|------|
| `lv_event_get_code(e)` | `lv_event_code_t` | 事件码 |
| `lv_event_get_current_target(e)` | `lv_obj_t*` | 当前处理事件的控件 |
| `lv_event_get_target(e)` | `lv_obj_t*` | 原始触发目标 |
| `lv_event_get_target_obj(e)` | `lv_obj_t*` | 原始触发控件（C++推荐） |
| `lv_event_get_user_data(e)` | `void*` | 注册时的user_data |
| `lv_event_get_param(e)` | `void*` | 手动发送时的参数 |
| `lv_event_get_prev_state(e)` | `lv_state_t` | 状态改变前的状态 |

> **C++提示**：当已知目标是控件时，优先使用 `lv_event_get_target_obj()`，无需类型转换。
> **警告**：仅在目标是控件时调用 `lv_event_get_target_obj()`，对Display或Indev调用是未定义行为。

## 四、事件冒泡与渗透

### 4.1 事件冒泡 (Event Bubbling)

事件从子控件向上传递给父控件：

```c
// 启用事件冒泡
lv_obj_add_flag(widget, LV_OBJ_FLAG_EVENT_BUBBLE);
```

**冒泡链**：
```
子控件触发事件
    │
    ▼
父控件处理事件（如果注册了回调）
    │
    ▼
祖父控件处理事件（如果启用了冒泡）
    │
    ▼
   ...
```

**获取原始触发控件**：
```c
static void parent_event_cb(lv_event_t * e)
{
    // 在父控件的回调中，获取原始子控件
    lv_obj_t * original_target = lv_event_get_target_obj(e);
}
```

### 4.2 事件渗透 (Event Trickle)

事件从父控件向下传递给子控件：

```c
// 启用事件渗透
lv_obj_add_flag(widget, LV_OBJ_FLAG_EVENT_TRICKLE);
```

**特点**：
- 仅影响直接子控件，不传递到孙控件
- 如需深层传递，每个子控件都需启用 `LV_OBJ_FLAG_EVENT_TRICKLE`
- 同样使用 `lv_event_get_target_obj()` 获取原始触发控件

## 五、手动发送事件

### 事件机制核心概念

理解LVGL事件机制的两个核心操作：

```
┌─────────────────────────────────────────────────────────────────┐
│                     LVGL 事件机制流程图                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   1. 添加事件 (add_event_cb)                                     │
│   ┌──────────────┐                                              │
│   │  lv_obj_add  │  ──→  注册回调函数到控件                      │
│   │ _event_cb()  │      "有事找我，这是我的回调函数"              │
│   └──────────────┘                                              │
│            │                                                    │
│            ▼                                                    │
│   ┌──────────────────────────────────────┐                     │
│   │           控 件 注 册 表              │                     │
│   │  ┌──────────────┬─────────────────┐  │                     │
│   │  │  事件类型     │  回调函数        │  │                     │
│   │  ├──────────────┼─────────────────┤  │                     │
│   │  │ LV_EVENT_XXX │ my_callback()   │  │                     │
│   │  └──────────────┴─────────────────┘  │                     │
│   └──────────────────────────────────────┘                     │
│                                                                 │
│   2. 触发事件（两种方式）                                         │
│   ┌─────────────────┐    ┌─────────────────┐                   │
│   │  A. 自动触发     │    │  B. 代码触发     │                   │
│   │  (用户操作)      │    │  (send_event)   │                   │
│   │                 │    │                 │                   │
│   │  用户点击按钮 ──→│←───  lv_obj_send   │                   │
│   │  用户滑动滑块 ──→│←───  _event()      │                   │
│   │  值发生变化  ──→ │    │                 │                   │
│   └─────────────────┘    └─────────────────┘                   │
│            │                      │                            │
│            └──────────┬───────────┘                            │
│                       ▼                                        │
│              ┌─────────────────┐                               │
│              │  调用回调函数    │                               │
│              │ my_callback(e)  │                               │
│              └─────────────────┘                               │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**一句话总结**：
- `add_event_cb` = **订阅通知**（留电话号码）
- `send_event` = **手动发通知**（自己打电话）

| 操作 | 作用 | 谁发起 | 类比 |
|------|------|--------|------|
| `lv_obj_add_event_cb()` | 注册回调函数 | 你的代码 | 留电话号码："有事打给我" |
| 用户操作触发 | 自动调用回调 | LVGL内部 | 别人打电话给你 |
| `lv_obj_send_event()` | 手动触发回调 | 你的代码 | 自己打电话给自己 |

### 5.1 发送事件到控件

```c
// 模拟按钮按下，手动触发事件
uint32_t btn_id = 0;
lv_obj_send_event(mbox, LV_EVENT_VALUE_CHANGED, &btn_id);
```

### 5.2 发送事件到显示设备和输入设备

```c
lv_display_send_event(disp, LV_EVENT_RESOLUTION_CHANGED, &new_resolution);
lv_indev_send_event(indev, LV_EVENT_CLICKED, NULL);
```

| 函数                                         | 说明        |
| ------------------------------------------ | --------- |
| `lv_obj_send_event(obj, code, param)`      | 发送事件到控件   |
| `lv_display_send_event(disp, code, param)` | 发送事件到显示设备 |
| `lv_indev_send_event(indev, code, param)`  | 发送事件到输入设备 |

### 5.3 自定义事件

```c
// 注册自定义事件ID
uint32_t MY_EVENT_1 = lv_event_register_id();

// 发送自定义事件
lv_obj_send_event(widget, MY_EVENT_1, &some_data);
```

### 5.4 LV_EVENT_REFRESH 特殊用途

`LV_EVENT_REFRESH` 用于通知控件刷新自身：

- 标签根据变量刷新文本（如当前时间）
- 语言切换时刷新标签
- 条件满足时启用按钮（如正确PIN码输入）
- 超出限制时添加/移除样式

## 六、完整示例

### 示例1：按钮点击计数器

```c
#include "lvgl.h"

static int click_count = 0;
static lv_obj_t * count_label;

static void btn_event_cb(lv_event_t * e)
{
    lv_event_code_t code = lv_event_get_code(e);

    if(code == LV_EVENT_CLICKED) {
        click_count++;
        lv_label_set_text_fmt(count_label, "Clicks: %d", click_count);
    }
}

void create_click_counter(void)
{
    lv_obj_t * btn = lv_button_create(lv_screen_active());
    lv_obj_set_size(btn, 120, 50);
    lv_obj_center(btn);
    lv_obj_add_event_cb(btn, btn_event_cb, LV_EVENT_CLICKED, NULL);

    lv_obj_t * label = lv_label_create(btn);
    lv_label_set_text(label, "Click Me");
    lv_obj_center(label);

    count_label = lv_label_create(lv_screen_active());
    lv_label_set_text(count_label, "Clicks: 0");
    lv_obj_align(count_label, LV_ALIGN_BOTTOM_MID, 0, -20);
}
```

### 示例2：事件冒泡处理多个按钮

```c
#include "lvgl.h"

static void parent_event_cb(lv_event_t * e)
{
    lv_event_code_t code = lv_event_get_code(e);
    lv_obj_t * target = lv_event_get_target_obj(e);

    if(code == LV_EVENT_CLICKED) {
        // 获取按钮的文本
        lv_obj_t * label = lv_obj_get_child(target, 0);
        const char * txt = lv_label_get_text(label);
        LV_LOG_USER("Button '%s' clicked", txt);
    }
}

void create_button_group(void)
{
    // 创建父容器，启用事件冒泡
    lv_obj_t * cont = lv_obj_create(lv_screen_active());
    lv_obj_set_size(cont, 200, 150);
    lv_obj_center(cont);
    lv_obj_set_flex_flow(cont, LV_FLEX_FLOW_COLUMN);
    lv_obj_add_flag(cont, LV_OBJ_FLAG_EVENT_BUBBLE);
    lv_obj_add_event_cb(cont, parent_event_cb, LV_EVENT_CLICKED, NULL);

    // 创建多个按钮（自动冒泡到父容器）
    const char * btn_texts[] = {"Button 1", "Button 2", "Button 3"};
    for(int i = 0; i < 3; i++) {
        lv_obj_t * btn = lv_button_create(cont);
        lv_obj_set_size(btn, 150, 30);
        lv_obj_add_flag(btn, LV_OBJ_FLAG_EVENT_BUBBLE);  // 启用冒泡

        lv_obj_t * label = lv_label_create(btn);
        lv_label_set_text(label, btn_texts[i]);
        lv_obj_center(label);
    }
}
```

### 示例3：长按与短按区分

```c
#include "lvgl.h"

static void advanced_btn_cb(lv_event_t * e)
{
    lv_event_code_t code = lv_event_get_code(e);
    lv_obj_t * btn = lv_event_get_target(e);

    switch(code) {
        case LV_EVENT_PRESSED:
            lv_obj_set_style_bg_color(btn, lv_palette_main(LV_PALETTE_RED), 0);
            break;

        case LV_EVENT_SHORT_CLICKED:
            LV_LOG_USER("Short click - toggle mode");
            break;

        case LV_EVENT_LONG_PRESSED:
            LV_LOG_USER("Long press - enter settings");
            lv_obj_set_style_bg_color(btn, lv_palette_main(LV_PALETTE_BLUE), 0);
            break;

        case LV_EVENT_RELEASED:
            lv_obj_set_style_bg_color(btn, lv_palette_main(LV_PALETTE_GREY), 0);
            break;

        default:
            break;
    }
}

void create_advanced_button(void)
{
    lv_obj_t * btn = lv_button_create(lv_screen_active());
    lv_obj_set_size(btn, 150, 60);
    lv_obj_center(btn);

    // 注册多个事件
    lv_obj_add_event_cb(btn, advanced_btn_cb, LV_EVENT_PRESSED, NULL);
    lv_obj_add_event_cb(btn, advanced_btn_cb, LV_EVENT_SHORT_CLICKED, NULL);
    lv_obj_add_event_cb(btn, advanced_btn_cb, LV_EVENT_LONG_PRESSED, NULL);
    lv_obj_add_event_cb(btn, advanced_btn_cb, LV_EVENT_RELEASED, NULL);

    lv_obj_t * label = lv_label_create(btn);
    lv_label_set_text(label, "Hold/Click");
    lv_obj_center(label);
}

### 示例4：滑块事件优化（避免频繁触发）

**问题场景**：滑块使用 `LV_EVENT_VALUE_CHANGED` 时，每次滑动都会触发回调，快速滑动可能导致回调堆积卡死。

**优化方案**：使用 `LV_EVENT_RELEASED` 只在释放时触发，或结合 `LV_EVENT_PRESSED` 更新显示 + `LV_EVENT_RELEASED` 执行操作。

```c
#include "lvgl.h"

static lv_obj_t * slider_value_label;
static int32_t final_value = 0;

// 方案1：只在释放时执行耗时操作
static void slider_efficient_cb(lv_event_t * e)
{
    lv_event_code_t code = lv_event_get_code(e);
    lv_obj_t * slider = lv_event_get_target(e);

    if(code == LV_EVENT_VALUE_CHANGED) {
        // 只更新显示，不执行耗时操作
        int32_t value = lv_slider_get_value(slider);
        lv_label_set_text_fmt(slider_value_label, "Value: %d", value);
    }
    else if(code == LV_EVENT_RELEASED) {
        // 用户松开时才执行耗时操作
        final_value = lv_slider_get_value(slider);
        LV_LOG_USER("Final value: %d - executing heavy operation...", final_value);
        // 执行网络请求、保存到Flash等耗时操作
    }
}

// 方案2：使用点击事件完全避免滑动触发
static void slider_click_only_cb(lv_event_t * e)
{
    lv_event_code_t code = lv_event_get_code(e);
    lv_obj_t * slider = lv_event_get_target(e);

    if(code == LV_EVENT_CLICKED) {
        // 只在点击（轻触）时触发，滑动不触发
        int32_t value = lv_slider_get_value(slider);
        lv_label_set_text_fmt(slider_value_label, "Set to: %d", value);
        LV_LOG_USER("Slider clicked, value: %d", value);
    }
}

void create_optimized_slider(void)
{
    // 创建标签显示当前值
    slider_value_label = lv_label_create(lv_screen_active());
    lv_label_set_text(slider_value_label, "Value: 50");
    lv_obj_align(slider_value_label, LV_ALIGN_TOP_MID, 0, 20);

    // 创建滑块 - 方案1：实时显示 + 释放时执行
    lv_obj_t * slider1 = lv_slider_create(lv_screen_active());
    lv_obj_set_size(slider1, 200, 20);
    lv_obj_align(slider1, LV_ALIGN_CENTER, 0, -30);
    lv_slider_set_value(slider1, 50, LV_ANIM_OFF);

    // 注册两个事件：VALUE_CHANGED用于显示，RELEASED用于执行
    lv_obj_add_event_cb(slider1, slider_efficient_cb, LV_EVENT_VALUE_CHANGED, NULL);
    lv_obj_add_event_cb(slider1, slider_efficient_cb, LV_EVENT_RELEASED, NULL);

    // 创建说明标签
    lv_obj_t * hint1 = lv_label_create(lv_screen_active());
    lv_label_set_text(hint1, "Mode 1: Drag to preview, release to apply");
    lv_obj_align_to(hint1, slider1, LV_ALIGN_OUT_TOP_MID, 0, -10);

    // 创建第二个滑块 - 方案2：仅点击触发
    lv_obj_t * slider2 = lv_slider_create(lv_screen_active());
    lv_obj_set_size(slider2, 200, 20);
    lv_obj_align(slider2, LV_ALIGN_CENTER, 0, 40);
    lv_slider_set_value(slider2, 50, LV_ANIM_OFF);
    lv_obj_add_event_cb(slider2, slider_click_only_cb, LV_EVENT_CLICKED, NULL);

    lv_obj_t * hint2 = lv_label_create(lv_screen_active());
    lv_label_set_text(hint2, "Mode 2: Click only (drag won't trigger)");
    lv_obj_align_to(hint2, slider2, LV_ALIGN_OUT_TOP_MID, 0, -10);
}
```

**对比总结**：

| 事件选择 | 触发时机 | 适用场景 | 性能影响 |
|----------|----------|----------|----------|
| `LV_EVENT_VALUE_CHANGED` | 每次值变化 | 需要实时反馈（如调节亮度预览） | 高频触发，需谨慎处理 |
| `LV_EVENT_RELEASED` | 松开时 | 执行耗时操作（如保存设置） | 低频触发，性能友好 |
| `LV_EVENT_CLICKED` | 点击/轻触 | 精确设置，滑动无效 | 最低频，最省资源 |
| 组合使用 | 实时显示+释放执行 | 预览+确认模式 | 平衡体验与性能 |

### 示例5：按钮控制滑块（避免滑动触发）

**方案3**：使用 +/- 按钮精确控制滑块值，完全避免滑动事件触发。

```c
#include "lvgl.h"

typedef struct {
    lv_obj_t * slider;
    lv_obj_t * value_label;
    int32_t step;  // 步进值
} slider_control_t;

static slider_control_t slider_ctrl;

// 按钮点击回调
static void btn_control_cb(lv_event_t * e)
{
    lv_obj_t * btn = lv_event_get_target(e);
    
    // 通过按钮获取父容器
    lv_obj_t * parent = lv_obj_get_parent(btn);
    
    // 通过父容器获取子控件
    // 假设布局: [减按钮][滑块][加按钮][标签]
    // 索引:      0        1      2       3
    lv_obj_t * slider = lv_obj_get_child(parent, 1);
    lv_obj_t * label = lv_obj_get_child(parent, 3);
    
    // 获取当前值和范围
    int32_t current = lv_slider_get_value(slider);
    int32_t min = lv_slider_get_min_value(slider);
    int32_t max = lv_slider_get_max_value(slider);
    
    // 判断是加按钮还是减按钮（通过按钮文本或user_data）
    lv_obj_t * btn_label = lv_obj_get_child(btn, 0);
    const char * txt = lv_label_get_text(btn_label);
    
    int32_t new_val = current;
    if(txt[0] == '+') {
        new_val = current + slider_ctrl.step;
        if(new_val > max) new_val = max;
    } else {
        new_val = current - slider_ctrl.step;
        if(new_val < min) new_val = min;
    }
    
    // 设置新值（不会触发VALUE_CHANGED事件，如果未注册的话）
    lv_slider_set_value(slider, new_val, LV_ANIM_ON);
    
    // 更新显示
    lv_label_set_text_fmt(label, "%d", new_val);
    
    LV_LOG_USER("Slider value changed to: %d", new_val);
    // 在这里执行耗时操作（如保存到Flash）
}

// 方案4：使用user_data传递控件引用（更推荐）
static void btn_control_v2_cb(lv_event_t * e)
{
    // 从user_data获取控件结构体
    slider_control_t * ctrl = (slider_control_t *)lv_event_get_user_data(e);
    lv_obj_t * btn = lv_event_get_target(e);
    
    lv_obj_t * btn_label = lv_obj_get_child(btn, 0);
    const char * txt = lv_label_get_text(btn_label);
    
    int32_t current = lv_slider_get_value(ctrl->slider);
    int32_t min = lv_slider_get_min_value(ctrl->slider);
    int32_t max = lv_slider_get_max_value(ctrl->slider);
    
    int32_t new_val = current;
    if(txt[0] == '+') {
        new_val = current + ctrl->step;
        if(new_val > max) new_val = max;
    } else {
        new_val = current - ctrl->step;
        if(new_val < min) new_val = min;
    }
    
    lv_slider_set_value(ctrl->slider, new_val, LV_ANIM_ON);
    lv_label_set_text_fmt(ctrl->value_label, "%d", new_val);
}

void create_button_controlled_slider(void)
{
    // 创建父容器

**三种方案对比**：

| 方案 | 实现方式 | 优点 | 缺点 | 适用场景 |
|------|----------|------|------|----------|
| **方案1** | `VALUE_CHANGED` + `RELEASED` | 保留滑动体验 | 仍需处理滑动事件 | 需要实时预览的场景 |
| **方案2** | 仅 `CLICKED` | 简单直接 | 滑动完全无效 | 精确点击设置 |
| **方案3** | +/- 按钮控制 | 精确控制、无滑动事件 | 操作稍繁琐 | 需要精确步进控制 |
| **方案4** | user_data 传递引用 | 代码清晰、可复用 | 需要额外结构体 | 多个滑块组管理 |

**父子控件获取方法**：

| 函数 | 说明 | 示例 |
|------|------|------|
| `lv_obj_get_parent(obj)` | 获取父控件 | `lv_obj_t * parent = lv_obj_get_parent(btn);` |
| `lv_obj_get_child(parent, index)` | 获取指定索引子控件 | `lv_obj_t * slider = lv_obj_get_child(parent, 1);` |
| `lv_obj_get_child_count(parent)` | 获取子控件数量 | `uint32_t count = lv_obj_get_child_count(parent);` |
| `lv_obj_get_index(obj)` | 获取控件在父容器中的索引 | `uint32_t idx = lv_obj_get_index(btn);` |

> **💡 提示**：通过 `get_parent` + `get_child` 的方式适合简单的父子关系。对于复杂界面，推荐使用 **方案4** 通过 `user_data` 传递结构体引用，代码更清晰且易于维护。

### 事件最佳实践

#### 1. 高频事件性能优化

对于滑块、滚轮等高频触发控件：

```c
// ❌ 不推荐：在 VALUE_CHANGED 中执行耗时操作
static void bad_slider_cb(lv_event_t * e) {
    int32_t val = lv_slider_get_value(lv_event_get_target(e));
    save_to_flash(val);        // 每次滑动都写Flash！
    send_to_server(val);       // 每次滑动都发网络请求！
}

// ✅ 推荐：使用 RELEASED 事件
static void good_slider_cb(lv_event_t * e) {
    lv_event_code_t code = lv_event_get_code(e);
    if(code == LV_EVENT_VALUE_CHANGED) {
        // 只更新UI，不执行耗时操作
        update_label(val);
    }
    else if(code == LV_EVENT_RELEASED) {
        // 用户松手时才执行耗时操作
        save_to_flash(val);
        send_to_server(val);
    }
}
```

#### 2. 事件选择决策表

| 控件类型 | 需要实时反馈 | 推荐事件 | 说明 |
|----------|-------------|----------|------|
| 滑块 | 是（如亮度预览）| `VALUE_CHANGED` + `RELEASED` | 实时更新显示，松手保存 |
| 滑块 | 否（如设置参数）| `RELEASED` 或 `CLICKED` | 避免频繁触发 |
| 开关 | - | `VALUE_CHANGED` | 状态切换不频繁 |
| 按钮 | - | `CLICKED` | 标准点击行为 |
| 文本输入 | 是 | `VALUE_CHANGED` | 实时搜索/验证 |
| 文本输入 | 否 | `READY` / `DEFOCUSED` | 输入完成后处理 |

#### 3. 避免回调中的坑

```c
// ❌ 错误：回调中阻塞操作
static void event_cb(lv_event_t * e) {
    vTaskDelay(1000);  // 阻塞整个UI！
}

// ✅ 正确：使用异步处理
static void event_cb(lv_event_t * e) {
    // 发送消息到队列，由任务处理
    xQueueSend(work_queue, &data, 0);
}

// ❌ 错误：回调中修改样式（绘制事件中）
static void draw_event_cb(lv_event_t * e) {
    lv_obj_set_style_bg_color(obj, red, 0);  // 断言失败！
}

// ✅ 正确：使用动画或延迟刷新
static void draw_event_cb(lv_event_t * e) {
    lv_obj_invalidate(obj);  // 标记重绘即可
}
```

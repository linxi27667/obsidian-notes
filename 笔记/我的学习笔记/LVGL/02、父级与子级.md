# LVGL 父级与子级关系 （长子如父 子级在上 父级在下）

## 核心概念

### 父子层级关系
- **子级显示在父级上面** - 子对象会覆盖父对象的显示区域
- **通过父级可以获取子级** - 使用下标方式遍历获取子对象
- **不推荐用全局变量** - 将 20 个 widget 作为函数参数传递不是理想做法
- **推荐** - 下标方式寻找子级

---

## 一、对象树结构

LVGL 采用树形结构管理对象，每个对象都有唯一的父对象和多个子对象。

```
屏幕 (Screen)
  │
  ├── 容器 (Container)
  │     ├── 按钮1 (Button)
  │     ├── 按钮2 (Button)
  │     └── 标签 (Label)
  │
  └── 图片 (Image)
```

---

## 二、图层概念（Layer）

### 2.1 图层基本原理

LVGL 通过**父子层级 + 创建顺序**共同决定对象的显示层次。

```
图层示意图（从底到顶）：
┌─────────────────────────┐  ← 最顶层（索引 -1）
│   child_3 (最后创建)     │
├─────────────────────────┤
│   child_2                │
├─────────────────────────┤
│   child_1 (最先创建)     │  ← 最底层（索引 0）
├─────────────────────────┤
│   parent (父对象)         │  ← 父对象在所有子对象之下
└─────────────────────────┘
```

---

### 2.2 图层显示规则

| 规则 | 说明 |
|------|------|
| **子级在上，父级在下** | 所有子对象都显示在父对象之上 |
| **创建顺序决定层次** | 先创建的对象在底层，后创建的在顶层 |
| **索引 0 = 最底层** | 第一个创建的子对象，被其他子对象覆盖 |
| **索引 -1 = 最顶层** | 最后创建的子对象，覆盖所有其他子对象 |

**示例**:
```c
lv_obj_t * parent = lv_obj_create(lv_scr_act());

lv_obj_t * child1 = lv_obj_create(parent);  // 索引 0，最底层
lv_obj_t * child2 = lv_obj_create(parent);  // 索引 1，中间层
lv_obj_t * child3 = lv_obj_create(parent);  // 索引 2 (= -1)，最顶层

// 显示顺序: parent → child1 → child2 → child3
// child3 会覆盖 child2 和 child1，child2 会覆盖 child1
```

---

### 2.3 图层与父子层级的关系

**父子嵌套时的图层叠加**:
```
屏幕层 (lv_scr_act)
  │
  ├── 父容器A (图层 L1)
  │     ├── 子对象A1 (图层 L1.0 - 底)
  │     ├── 子对象A2 (图层 L1.1 - 中)
  │     └── 子对象A3 (图层 L1.2 - 顶)
  │
  └── 父容器B (图层 L2)
        └── 子对象B1 (图层 L2.0)
```

**说明**:
- 父容器A 和 父容器B 的图层关系由它们的创建顺序决定
- 子对象的图层仅在其父对象的区域内生效
- 不同父容器的子对象互不影响（各自独立渲染）

## 三、图层控制 API

### 3.1 移动到后台（最底层）

```c
lv_obj_move_background(obj);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t*` | 要移动的对象指针 |

**效果**: 将对象移动到其父对象中所有子对象的**最底层**（索引变为 0）

---

### 3.2 移动到前台（最顶层）

```c
lv_obj_move_foreground(obj);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t*` | 要移动的对象指针 |

**效果**: 将对象移动到其父对象中所有子对象的**最顶层**（索引变为 -1）

---

### 3.3 移动到指定索引位置

```c
lv_obj_move_to_index(obj, idx);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t*` | 要移动的对象指针 |
| `idx` | `int32_t` | 目标索引位置 |

**索引含义**:
| 索引值 | 图层位置 | 说明 |
|------|------|------|
| `0` | 最底层 | 被所有同级子对象覆盖 |
| `1` | 第二层 | 覆盖索引 0，被其他覆盖 |
| `-1` | 最顶层 | 覆盖所有同级子对象 |
| `-2` | 倒数第二层 | 仅被最顶层覆盖 |

---

### 3.4 交换两个对象的图层位置

```c
lv_obj_swap(obj1, obj2);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj1` | `lv_obj_t*` | 第一个对象指针 |
| `obj2` | `lv_obj_t*` | 第二个对象指针 |

**说明**: 
- 两个对象必须在**同一个父对象**中
- 交换后，两者的图层顺序互换

---

### 3.5 获取当前图层索引

```c
int32_t idx = lv_obj_get_index(obj);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t*` | 目标对象指针 |

**返回值**: `int32_t` - 对象在父级中的索引（0 = 最底层，-1 = 最顶层）

## 四、图层控制示例

### 示例1：点击按钮后移到顶层

```c
static void btn_click_cb(lv_event_t * e)
{
    lv_obj_t * btn = lv_event_get_target(e);
    
    // 点击后将按钮移到最顶层（覆盖其他按钮）
    lv_obj_move_foreground(btn);
}

// 创建多个重叠的按钮
lv_obj_t * btn1 = lv_btn_create(lv_scr_act());
lv_obj_set_pos(btn1, 0, 0);
lv_obj_set_size(btn1, 100, 100);

lv_obj_t * btn2 = lv_btn_create(lv_scr_act());
lv_obj_set_pos(btn2, 20, 20);
lv_obj_set_size(btn2, 100, 100);

lv_obj_t * btn3 = lv_btn_create(lv_scr_act());
lv_obj_set_pos(btn3, 40, 40);
lv_obj_set_size(btn3, 100, 100);

// 添加点击事件
lv_obj_add_event_cb(btn1, btn_click_cb, LV_EVENT_CLICKED, NULL);
lv_obj_add_event_cb(btn2, btn_click_cb, LV_EVENT_CLICKED, NULL);
lv_obj_add_event_cb(btn3, btn_click_cb, LV_EVENT_CLICKED, NULL);
```

---

### 示例2：图层顺序动态调整

```c
// 原始顺序: child1(底层) → child2 → child3(顶层)

// 将 child1 移到顶层
lv_obj_move_foreground(child1);
// 新顺序: child2(底层) → child3 → child1(顶层)

// 将 child3 移到底层
lv_obj_move_background(child3);
// 新顺序: child3(底层) → child2 → child1(顶层)

// 交换 child2 和 child1
lv_obj_swap(child2, child1);
// 新顺序: child3(底层) → child1 → child2(顶层)
```

---

### 示例3：根据索引精确定位图层

```c
lv_obj_t * parent = lv_obj_create(lv_scr_act());

// 创建5个子对象
for(int i = 0; i < 5; i++) {
    lv_obj_t * child = lv_obj_create(parent);
    lv_obj_set_size(child, 50, 50);
}

// 将第一个创建的对象移到中间位置（索引 2）
lv_obj_move_to_index(lv_obj_get_child(parent, 0), 2);

// 将最后一个创建的对象移到第二层（索引 1）
lv_obj_move_to_index(lv_obj_get_child(parent, -1), 1);
```

## 六、设置父级

### 6.1 移动对象到新父级

```c
lv_obj_set_parent(widget, new_parent);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t*` | 要移动的对象指针 |
| `parent` | `lv_obj_t*` | 新的父对象指针 |

**说明**:
- 对象会移动到新父对象中
- 移动后，对象会成为新父对象的**最顶层子对象**（最后一个子对象）
- 原父对象的子对象列表中会移除该对象

**示例**:
```c
lv_obj_t * old_parent = lv_obj_create(lv_scr_act());
lv_obj_t * new_parent = lv_obj_create(lv_scr_act());
lv_obj_t * child = lv_obj_create(old_parent);

// 将 child 从 old_parent 移动到 new_parent
lv_obj_set_parent(child, new_parent);
```

## 七、获取子级

### 7.1 通过索引获取子对象

```c
lv_obj_t * child = lv_obj_get_child(parent, idx);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `parent` | `lv_obj_t*` | 父对象指针 |
| `idx` | `int32_t` | 子对象索引 |

**返回值**: `lv_obj_t*` - 对应索引的子对象指针

**索引与图层对应关系**:
| 索引值 | 图层位置 | 创建顺序 |
|------|------|------|
| `0` | 最底层 | 第一个创建 |
| `1` | 第二层 | 第二个创建 |
| `-1` | 最顶层 | 最后创建 |
| `-2` | 倒数第二层 | 倒数第二个创建 |

---

### 7.2 通过类型过滤获取子对象

```c
lv_obj_t * child = lv_obj_get_child_by_type(parent, idx, &lv_slider_class);
```

| 参数        | 类型                      | 说明        |
| --------- | ----------------------- | --------- |
| `parent`  | `lv_obj_t*`             | 父对象指针     |
| `idx`     | `int32_t`               | 在过滤类型中的索引 |
| `class_p` | `const lv_obj_class_t*` | 要过滤的类型类指针 |

**说明**:
- 只返回指定类型的子对象
- 可用类型: `lv_slider_class`, `lv_btn_class`, `lv_label_class` 等

---

### 7.3 遍历所有子对象

```c
uint32_t i;
uint32_t count = lv_obj_get_child_count(parent);

for(i = 0; i < count; i++) {
    lv_obj_t * child = lv_obj_get_child(parent, i);
    /* 对 child 进行操作 */
}
```

| 函数 | 说明 |
|------|------|
| `lv_obj_get_child_count(parent)` | 返回子对象总数 |

## 八、查找子对象（命名系统）

### 8.1 通过名称查找

**前提**: 需在 `lv_conf.h` 中启用 `LV_USE_OBJ_NAME`

#### 查找直接子对象
```c
lv_obj_t * child = lv_obj_get_child_by_name(parent, "button");
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `parent` | `lv_obj_t*` | 父对象指针 |
| `name` | `const char*` | 子对象名称 |

**返回值**: `lv_obj_t*` - 匹配名称的子对象指针，未找到返回 `NULL`

#### 通过路径查找嵌套子对象
```c
lv_obj_t * child = lv_obj_get_child_by_name(cont, "buttons/mybtn_2");
```

**说明**:
- 使用 `/` 分隔路径层级
- `"buttons/mybtn_2"` 表示: cont 的子对象 buttons 的子对象 mybtn_2

---

### 8.2 深度查找（任意层级）

```c
lv_obj_t * widget = lv_obj_find_by_name(parent, "mybtn_1");
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `parent` | `lv_obj_t*` | 搜索起始父对象 |
| `name` | `const char*` | 要查找的对象名称 |

**说明**:
- 在 parent 的所有后代中搜索（不限于直接子对象）
- 适用于忽略层级结构、仅按名称搜索的场景

## 九、对象命名系统

### 9.1 设置对象名称

```c
lv_obj_set_name(obj, "my_button");
lv_obj_set_name_static(obj, "my_button");  // 静态版本
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t*` | 目标对象指针 |
| `name` | `const char*` | 名称字符串 |

**两种版本区别**:
| 函数 | 说明 |
|------|------|
| `lv_obj_set_name()` | LVGL 会分配内存存储名称副本 |
| `lv_obj_set_name_static()` | 只存储指针，名称字符串必须保持有效 |

---

### 9.2 自动命名规则

| 规则 | 结果 |
|------|------|
| 未设置名称 | 默认为 `<widget_type>_#`，如 `lv_button_0` |
| 名称以 `#` 结尾 | 自动添加索引，如 `mybtn_#` → `mybtn_0`, `mybtn_1` |
| 自定义名称 | 使用指定名称 |

**命名示例**:
```
container "cont"
  │
  ├── container "header"
  │     ├── label (未命名) → "lv_label_0"
  │     ├── label "title" → "title"
  │     └── label (未命名) → "lv_label_1"
  │
  └── container "buttons"
  │     ├── button (未命名) → "lv_button_0"
  │     ├── button "second_button" → "second_button"
  │     ├── button (未命名) → "lv_button_1"
  │     ├── button "lv_button_#" → "lv_button_2"
  │     ├── button "mybtn_#" → "mybtn_0"
```

**注意**: 索引是"流动的"，如果 `mybtn_2` 被删除，原来的 `mybtn_3` 会变为 `mybtn_2`

### 示例1：图层叠叠乐效果

```c
// 创建3个重叠的彩色方块，点击任意方块后移到顶层
void create_stackable_boxes(void)
{
    lv_obj_t * parent = lv_obj_create(lv_scr_act());
    lv_obj_set_size(parent, 200, 200);
    lv_obj_center(parent);

    lv_obj_set_pos(box1, 0, 0);
    lv_obj_set_style_bg_color(box1, lv_palette_main(LV_PALETTE_RED), 0);
    
    lv_obj_t * box2 = lv_obj_create(parent);
    lv_obj_set_size(box2, 80, 80);
    lv_obj_set_pos(box2, 30, 30);
    lv_obj_set_style_bg_color(box2, lv_palette_main(LV_PALETTE_GREEN), 0);
    
    lv_obj_t * box3 = lv_obj_create(parent);
    lv_obj_set_size(box3, 80, 80);
    lv_obj_set_pos(box3, 60, 60);
    lv_obj_set_style_bg_color(box3, lv_palette_main(LV_PALETTE_BLUE), 0);
    
    // 默认显示顺序: box1(底) → box2 → box3(顶)

    // 添加点击事件：点击后移到顶层

    lv_obj_add_event_cb(box2, btn_click_cb, LV_EVENT_CLICKED, NULL);
```

```c
void create_layer_panel(void)
```

## 注意事项与最佳实践

### 父子关系管理

| 注意点 | 说明 | 建议 |
|--------|------|------|
| **层级深度** | 嵌套层级过多影响性能 | 建议不超过5-6层 |
| **删除父对象** | 会级联删除所有子对象 | 确保不需要子对象后再删除父对象 |
| **改变父对象** | 使用 `lv_obj_set_parent()` 可改变父子关系 | 对象会保持原有坐标，但相对于新父对象 |
| **循环引用** | 避免父子关系形成循环 | 会导致内存泄漏或崩溃 |

### 图层控制注意事项

```c
// ❌ 错误：频繁移动图层导致性能问题
for(int i = 0; i < 100; i++) {
    lv_obj_move_foreground(obj);
}

// ✅ 正确：只在必要时移动图层
if(need_on_top) {
    lv_obj_move_foreground(obj);
}
```

### 获取子对象注意事项

| 方法 | 适用场景 | 注意事项 |
|------|----------|----------|
| `lv_obj_get_child(parent, index)` | 已知索引位置 | 索引越界返回NULL |
| `lv_obj_get_child_count(parent)` | 遍历前获取数量 | 遍历过程中子对象数量可能变化 |
| `lv_obj_get_parent(obj)` | 获取父对象 | 屏幕对象没有父对象 |

### 常见问题

**Q: 子对象超出父对象边界会怎样？**
A: 默认情况下会显示在父对象外部。如需裁剪，设置 `LV_OBJ_FLAG_OVERFLOW_VISIBLE` 标志。

**Q: 如何确保某个对象始终在最上层？**
A: 每次创建新对象后调用 `lv_obj_move_foreground(obj)`。

**Q: 删除对象后其指针还能用吗？**
A: 不能！删除后应立即将指针设为NULL，避免悬空指针。

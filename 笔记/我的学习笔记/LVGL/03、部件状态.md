# LVGL 部件、部件与状态 （样式系统通过部件 + 状态的组合选择器精确控制 UI 元素的外观）

## 核心概念

### 部件（Parts）
- ==部件是控件的"物理组成部分"== - 每个控件由多个部件构成，如滑块 = 背景 + 指示条 + 旋钮
- ==`LV_PART_MAIN` 表示控件主体== - 所有控件都有 MAIN 部分，通常指背景或整体区域
- ==不同控件有不同的部件== - 滑块有 KNOB/INDICATOR，按钮通常只有 MAIN

### 状态（States）
- ==状态是控件的"交互模式"== - 如默认、按下、聚焦、禁用等
- ==状态可以叠加== - 通过位运算 `|` 组合，如 `LV_STATE_PRESSED | LV_STATE_FOCUSED`
- ==默认状态值为 0== - 不指定状态时就是默认状态

### 选择器（Selector）
- ==选择器 = 部件 | 状态== - 用位运算将部件和状态编码到一个值中
- ==`lv_obj_set_style_XXX(obj, value, selector)`== - 第三个参数 selector 决定了样式应用到哪个部位、哪种状态下

## 一、部件系统详解

### 1.1 通用部件（所有控件可用）

| 部件常量 | 说明 | 示例 |
|----------|------|------|
| `LV_PART_MAIN` | 控件主体/背景区域 | 按钮整体、滑块背景槽 |
| `LV_PART_SCROLLBAR` | 滚动条 | 内容超出时显示的滚动条 |
| `LV_PART_INDICATOR` | 指示器 | 进度条的已填充部分、滑块的指示条 |
| `LV_PART_KNOB` | 旋钮/拖动手柄 | 滑块的圆形拖拽点 |
| `LV_PART_SELECTED` | 选中区域 | 下拉列表中被选中的项 |

### 1.2 滑块的部件结构

```
┌─────────────────────────────────────┐  ← 滑块整体对象
│  ┌─────────────────────────────┐    │
│  │   LV_PART_MAIN (背景槽)       │    │  ← 深色凹槽
│  │  ┌────────────┐  ●           │    │
│  │  │LV_PART_INDICATOR│LV_PART_KNOB│  │
│  │  │ (已填充区域)  │ (圆形旋钮)   │    │
│  │  └────────────┘              │    │
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```

**说明**:
- `LV_PART_MAIN` — 滑块的背景槽（未填充部分 + 已填充部分的容器）
- `LV_PART_INDICATOR` — 已填充的进度区域（从最小值到当前值）
- `LV_PART_KNOB` — 圆形拖拽旋钮

### 1.3 滑块部件样式 API

```c
// 修改滑块背景槽颜色
lv_obj_set_style_bg_color(slider, lv_palette_main(LV_PALETTE_GREY), LV_PART_MAIN);

// 修改指示条（已填充区域）颜色
lv_obj_set_style_bg_color(slider, lv_palette_main(LV_PALETTE_BLUE), LV_PART_INDICATOR);

// 修改旋钮颜色
lv_obj_set_style_bg_color(slider, lv_palette_main(LV_PALETTE_RED), LV_PART_KNOB);

// 修改旋钮大小
lv_obj_set_style_size(slider, lv_pct(140), LV_PART_KNOB);

// 修改指示条圆角
lv_obj_set_style_radius(slider, 5, LV_PART_INDICATOR);

// 修改滚动条颜色（内容超出时显示）
lv_obj_set_style_bg_color(obj, lv_palette_main(LV_PALETTE_ORANGE), LV_PART_SCROLLBAR);
```

**API 速查**:
| 函数 | 作用部件 | 说明 |
|------|----------|------|
| `lv_obj_set_style_bg_color(obj, color, part)` | 任意部件 | 设置背景颜色 |
| `lv_obj_set_style_bg_opa(obj, opa, part)` | 任意部件 | 设置背景透明度 |
| `lv_obj_set_style_radius(obj, radius, part)` | 任意部件 | 设置圆角半径 |
| `lv_obj_set_style_border_width(obj, width, part)` | 任意部件 | 设置边框宽度 |
| `lv_obj_set_style_border_color(obj, color, part)` | 任意部件 | 设置边框颜色 |
| `lv_obj_set_style_pad_all(obj, pad, part)` | 任意部件 | 设置四周内边距 |
| `lv_obj_set_style_size(obj, size, part)` | KNOB 等 | 设置部件尺寸（百分比或像素） |

## 二、状态系统详解

### 2.1 状态枚举值

| 状态常量 | 值 | 说明 |
|----------|-----|------|
| `LV_STATE_DEFAULT` | 0x0000 | 默认状态（无交互） |
| `LV_STATE_CHECKED` | 0x0001 | 已勾选（用于开关、复选框） |
| `LV_STATE_FOCUSED` | 0x0002 | 获得焦点 |
| `LV_STATE_FOCUS_KEY` | 0x0004 | 键盘焦点 |
| `LV_STATE_EDITED` | 0x0008 | 正在编辑 |
| `LV_STATE_HOVERED` | 0x0010 | 鼠标悬停 |
| `LV_STATE_PRESSED` | 0x0020 | 被按下 |
| `LV_STATE_SCROLLED` | 0x0040 | 正在滚动 |
| `LV_STATE_DISABLED` | 0x0080 | 禁用状态 |

**注意**: `LV_STATE_DEFAULT` 值为 0，所以不需要显式传入。

### 2.2 为什么用位运算 `|` 组合？

```c
// 你困惑的写法：
lv_obj_set_style_bg_color(btn, lv_palette_main(LV_PALETTE_RED), LV_PART_MAIN | LV_STATE_PRESSED);
```

**原因**: LVGL 的选择器是一个 **32 位整数**，高位存部件编号、低位存状态编号。用位运算 `|` 把它们编码到同一个值中：

```
LV_PART_MAIN      = 0x00000000  (部件编号在高位)
LV_STATE_PRESSED  = 0x00000020  (状态编号在低位)
─────────────────────────────────
LV_PART_MAIN | LV_STATE_PRESSED = 0x00000020  (组合选择器)
```

**这表示**: "当按钮的 MAIN 部件处于 PRESSED 状态时，使用这个样式"。

### 2.3 常见组合示例

```c
// 默认状态下按钮主体为蓝色
lv_obj_set_style_bg_color(btn, lv_palette_main(LV_PALETTE_BLUE), LV_PART_MAIN);
// 等价于（因为 LV_STATE_DEFAULT = 0）:
lv_obj_set_style_bg_color(btn, lv_palette_main(LV_PALETTE_BLUE), LV_PART_MAIN | LV_STATE_DEFAULT);

// 按下时按钮主体变为红色
lv_obj_set_style_bg_color(btn, lv_palette_main(LV_PALETTE_RED), LV_PART_MAIN | LV_STATE_PRESSED);

// 聚焦时边框变绿色
lv_obj_set_style_border_color(btn, lv_palette_main(LV_PALETTE_GREEN), LV_PART_MAIN | LV_STATE_FOCUSED);

// 禁用时背景变灰色
lv_obj_set_style_bg_color(btn, lv_palette_main(LV_PALETTE_GREY), LV_PART_MAIN | LV_STATE_DISABLED);

// 按下时旋钮（KNOB）变大
lv_obj_set_style_size(slider, lv_pct(150), LV_PART_KNOB | LV_STATE_PRESSED);

// 聚焦且按下时指示条变亮色
lv_obj_set_style_bg_color(slider, lv_palette_lighten(LV_PALETTE_BLUE, 2),
                          LV_PART_INDICATOR | LV_STATE_FOCUSED | LV_STATE_PRESSED);
```

## 三、`LV_PART_MAIN` 深入理解

### 3.1 什么是 `LV_PART_MAIN`？

`LV_PART_MAIN` 表示控件的**主体部分**，是最基础的部件。可以理解为"控件本身的背景/容器区域"。

| 控件 | `LV_PART_MAIN` 代表什么 |
|------|------------------------|
| 按钮 (Button) | 整个按钮的背景矩形 |
| 滑块 (Slider) | 背景槽（容纳指示条和旋钮的容器） |
| 开关 (Switch) | 开关的轨道/背景槽 |
| 复选框 (Checkbox) | 勾选框本身 |
| 标签 (Label) | 文字区域 |
| 容器 (Obj/Container) | 容器背景区域 |

### 3.2 简写形式

```c
// 以下两种写法等价：
lv_obj_set_style_bg_color(obj, color, 0);                    // 简写：selector = 0
lv_obj_set_style_bg_color(obj, color, LV_PART_MAIN);         // 完整写法

// 因为 LV_PART_MAIN = 0，LV_STATE_DEFAULT = 0，所以 0 | 0 = 0
```

**推荐**: 初学者写 `LV_PART_MAIN` 更易读，熟练后可以用 `0` 简写。

## 四、样式继承与优先级

### 4.1 样式优先级规则

当多个选择器匹配同一样式属性时，优先级从高到低：

1. **具体状态样式** > **默认状态样式**
   - `LV_STATE_PRESSED` 的样式会覆盖 `LV_STATE_DEFAULT`
2. **后设置的样式** > **先设置的样式**（同一选择器下）
3. **本地样式** > **继承样式**

```c
// 示例：按下状态覆盖默认
lv_obj_set_style_bg_color(btn, lv_palette_main(LV_PALETTE_BLUE), LV_PART_MAIN);        // 默认蓝色
lv_obj_set_style_bg_color(btn, lv_palette_main(LV_PALETTE_RED), LV_PART_MAIN | LV_STATE_PRESSED); // 按下变红色

// 用户没按下时 → 蓝色
// 用户按下时   → 红色（PRESSED 优先级高于 DEFAULT）
// 用户松开后   → 恢复蓝色
```

### 4.2 状态转换流程图

```
[默认状态] ──鼠标按下──→ [按下状态]
     ↑                      │
     │                  鼠标松开
     │                      ↓
     └────────────── 恢复默认
```

**说明**: 状态转换由 LVGL 自动管理，你只需要为不同状态设置样式即可。

## 五、位置与尺寸补充

### 5.1 百分比尺寸

```c
// 设置为父对象 50% 宽高
lv_obj_set_size(btn, LV_PCT(50), LV_PCT(50));

// 宽度 100%，高度固定
lv_obj_set_size(btn, LV_PCT(100), 40);
```

**说明**: `LV_PCT(x)` 是 `lv_pct(x)` 的宏定义，表示相对于父对象内容区的百分比。

### 5.2 对齐方式

```c
// 对齐到顶部中央，偏移 (0, 10)
lv_obj_align(btn, LV_ALIGN_TOP_MID, 0, 10);

// 对齐到正中央
lv_obj_align(btn, LV_ALIGN_CENTER, 0, 0);
// 可简写为：
lv_obj_center(btn);
```

**常用对齐枚举**:
| 枚举值 | 位置 |
|--------|------|
| `LV_ALIGN_CENTER` | 正中央 |
| `LV_ALIGN_TOP_MID` | 顶部中央 |
| `LV_ALIGN_TOP_LEFT` | 左上角 |
| `LV_ALIGN_TOP_RIGHT` | 右上角 |
| `LV_ALIGN_BOTTOM_MID` | 底部中央 |
| `LV_ALIGN_LEFT_MID` | 左侧中央 |
| `LV_ALIGN_RIGHT_MID` | 右侧中央 |

### 5.3 父子位置关系

- 父对象位置变化时，所有子对象跟随移动
- 子对象的坐标和大小相对于父对象的内容区（padding 内部）
- 子对象超出父对象的部分会被裁剪（不可见）

## 六、完整示例

### 示例1：自定义彩色滑块

```c
void create_custom_slider(void)
{
    lv_obj_t * slider = lv_slider_create(lv_scr_act());
    lv_obj_set_size(slider, 200, 10);
    lv_obj_center(slider);
    lv_slider_set_range(slider, 0, 100);

    // 默认状态 - 背景槽灰色
    lv_obj_set_style_bg_color(slider, lv_palette_main(LV_PALETTE_GREY), LV_PART_MAIN);
    // 默认状态 - 指示条蓝色
    lv_obj_set_style_bg_color(slider, lv_palette_main(LV_PALETTE_BLUE), LV_PART_INDICATOR);
    // 默认状态 - 旋钮白色
    lv_obj_set_style_bg_color(slider, lv_color_white(), LV_PART_KNOB);

    // 按下状态 - 指示条变亮蓝
    lv_obj_set_style_bg_color(slider, lv_palette_lighten(LV_PALETTE_BLUE, 2),
                              LV_PART_INDICATOR | LV_STATE_PRESSED);
    // 按下状态 - 旋钮变大
    lv_obj_set_style_size(slider, lv_pct(140), LV_PART_KNOB | LV_STATE_PRESSED);

    // 禁用状态 - 整体变灰
    lv_obj_set_style_bg_color(slider, lv_palette_main(LV_PALETTE_GREY),
                              LV_PART_INDICATOR | LV_STATE_DISABLED);
    lv_obj_set_style_bg_color(slider, lv_palette_main(LV_PALETTE_GREY),
                              LV_PART_KNOB | LV_STATE_DISABLED);
}
```

## 注意事项与最佳实践

### 部件使用注意事项

| 注意点 | 说明 | 建议 |
|--------|------|------|
| **部件有效性** | 不是所有控件都支持所有部件 | 查阅文档确认控件支持的部件 |
| **默认部件** | 所有控件都有 `LV_PART_MAIN` | 最简单的样式设置目标 |
| **部件叠加** | 部件有层级关系（如INDICATOR在MAIN之上） | 理解部件层级有助于调试显示问题 |
| **滚动条部件** | 只有内容可滚动的控件才有 | 需设置足够内容才会显示 |

### 状态管理注意事项

```c
// ❌ 错误：直接使用数字设置状态
lv_obj_add_state(btn, 0x01);  // 可读性差，易出错

// ✅ 正确：使用状态常量
lv_obj_add_state(btn, LV_STATE_PRESSED);

// ❌ 错误：同时添加冲突状态
lv_obj_add_state(btn, LV_STATE_DISABLED | LV_STATE_PRESSED);
// 禁用状态下按下状态可能无效

// ✅ 正确：根据逻辑合理设置状态
if(is_enabled) {
    lv_obj_clear_state(btn, LV_STATE_DISABLED);
} else {
    lv_obj_add_state(btn, LV_STATE_DISABLED);
}
```

### 样式优先级陷阱

| 优先级 | 来源 | 说明 |
|--------|------|------|
| 高 | 用户代码直接设置 | `lv_obj_set_style_XXX()` |
| 中 | 主题样式 | `lv_theme_default_init()` |
| 低 | 控件默认样式 | 控件创建时自带 |

**注意**：后设置的样式会覆盖先设置的，与优先级无关。

### 常见问题

**Q: 设置了样式但没有效果？**
A: 检查：1) 选择器是否正确（部件+状态）；2) 样式是否被后设置的覆盖；3) 控件是否支持该部件。

**Q: 如何查看控件当前所有状态？**
A: 使用 `lv_obj_get_state(obj)` 获取当前状态位掩码。

**Q: 禁用状态的控件还能响应事件吗？**
A: 取决于控件实现，通常禁用状态会阻止大部分交互事件。

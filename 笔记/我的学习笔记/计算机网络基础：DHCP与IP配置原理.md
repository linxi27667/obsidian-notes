# 计算机网络基础：为什么需要配置 IP、网关、DNS

---

## 一、DHCP 是什么

### 通俗理解

> [!NOTE] DHCP = 自动发门牌号的服务员
> 想象你住酒店：去前台登记，前台自动给你分配一个房间号（808）。你什么都不用管，拎包入住。
>
> 这个"自动分配房间号"的服务，就是 DHCP。

### 技术定义

- **DHCP = Dynamic Host Configuration Protocol**（动态主机配置协议）
- 作用：**自动给设备分配 IP 地址**，不用你手动填
- 谁在做：路由器、服务器、企业网络设备

### 什么情况下会没有 DHCP

| 环境 | 有 DHCP？ | 你要怎么做 |
|------|----------|-----------|
| 家里/办公室 | ✅ 有（路由器内置） | 插上网线，自动有 IP |
| 学校实验室 | ❌ 可能没有 | **手动配置静态 IP** |
| 工厂工控网络 | ❌ 通常没有 | **手动配置静态 IP** |
| 租用服务器 | ❌ 通常没有 | **手动配置静态 IP** |

> [!WARNING] 这次为什么 dhclient 卡死
> 因为目标网络**没有 DHCP 服务器**，所以客户端发广播请求后没人回应，一直等到超时。
> 这时必须放弃自动获取，**手动配置 IP**。

## 二、为什么需要配置这些

### 完整上网的"三层门"比喻

```
            ┌──────────────────────────────┐
  第三层     │  DNS（域名解析）               │
  "查电话簿"  │  220.181.38.150              │
            │  你说"baidu.com"               │
            │  它告诉你 IP 是多少              │
            └──────────────────────────────┘
                       ↓ 知道了 IP 才能联系
            ┌──────────────────────────────┐
  第二层     │  默认网关（出口大门）            │
  "出小区"   │  10.3.36.254                 │
            │  所有不是本地的包都从这出去        │
            └──────────────────────────────┘
                       ↓ 包从这里出小区
            ┌──────────────────────────────┐
  第一层     │  本机 IP 地址（门牌号）          │
  "我是谁"   │  10.3.36.213                │
            │  局域网内的唯一标识              │
            └──────────────────────────────┘
```

> [!TIP] 缺了任何一层会怎样？
> - 没配 IP → 快递没有寄件人地址，快递柜拒收
> - 没配网关 → 快递不知道往哪寄，出不了小区
> - 没配 DNS → 你只知道对方姓名，不知道手机号（IP），无法打电话

## 三、每个配置项详解

### 1. IP 地址

```
10.3.36.213/24
     │        │
     │        └── 子网掩码：255.255.255.0（前24位是网络号）
     └── 你的设备在网络上的"门牌号"
```

| 项目 | 值 | 含义 |
|------|-----|------|
| 本机 IP | `10.3.36.213` | 局域网内唯一标识你的电脑 |
| 子网掩码 | `/24` 或 `255.255.255.0` | 前24位相同 = 同一"街道"，可以直连 |
| 网段 | `10.3.36.x` | 同网段的设备可以直接通信 |

> [!NOTE] 为什么不能乱填 IP
> 如果你填了 `192.168.1.100`，但局域网是 `10.3.36.x`
> 那你就"住错了街道"，收不到任何同局域网设备的包

---

### 2. 默认网关（Gateway）

```
网关：10.3.36.254
       │
       └── 局域网的"大门"，所有外出包都经过这里
```

> [!NOTE] 网关的通俗理解
> 想象你住在一个小区（局域网），你想寄信到外省（公网）。
> 你把信交给**小区门卫**（网关），门卫帮你转发到外面。
> 没有门卫，你连小区都出不去。

- **同一网段的设备**（如 `10.3.36.x`）可以**直接互访**（靠交换机）
- **跨网段的设备**必须通过**网关**出去
- 没有网关 → 能 ping 通邻居（内网），ping 不通百度（外网）

---

### 3. DNS 服务器

```
DNS：223.5.5.5（阿里DNS） / 114.114.114.114（114DNS）
```

> [!NOTE] DNS 的通俗理解
> DNS = 网络的"电话簿"
>
> 你说"我要找 baidu.com" → DNS 查表告诉你"他的号码是 220.181.38.150"
> 你说"我要找 google.com" → DNS 查表告诉你"他的号码是 142.250.191.78"
>
> 没有 DNS？你只能记住每个网站的 IP 才能上网（没人做得到）

- **DNS 故障的典型表现**：
  ```
  ping 220.181.38.150  ✅ 通（直接用 IP）
  ping baidu.com       ❌ 失败（Name or service not known）
  ```
  → 说明 DNS 没配对，IP 能通但域名不能

## 四、完整排障流程图

> [!NOTE] 排障黄金法则：从内到外，一层层排查

```
1. 物理层正常吗？
   └─→ sudo ethtool enp1s0
       → Link detected: yes ✅ 继续
       → Link detected: no ❌ 检查网线

2. 内网（网关）通吗？
   └─→ ping 10.3.36.254
       → 通 ✅ 继续
       → 不通 ❌ 检查 IP 和网关配置

3. 外网（公网 IP）通吗？
   └─→ ping 223.5.5.5
       → 通 ✅ 继续
       → 不通 ❌ 网关配置错误

4. 域名解析（DNS）通吗？
   └─→ ping baidu.com
       → 通 ✅ 全部正常！
       → 不通 ❌ DNS 配置错误
```

## 五、高频问题解答

### Q：为什么家里不用配置这些？
> A：因为你家的**路由器内置了 DHCP 服务器**，插上网线后它自动给你分配 IP、子网掩码、网关、DNS，全部自动完成，你什么都不用管。

### Q：为什么学校/实验室/工厂需要手动配置？
> A：因为这些地方**没有 DHCP 服务器**，或者有但你不在它的分配范围内。所以必须手动填。

### Q：DHCP 和静态 IP 哪个好？
> A：看场景：
> - 家用/办公 → DHCP 省心
> - 服务器/工控设备 → 静态 IP 稳定（IP 固定不变，方便管理）

### Q：什么是公网 IP 和内网 IP？
> A：
> - **内网 IP**（如 `10.3.36.x`、`192.168.x.x`）：只能在局域网内用，不能直接被外网访问
> - **公网 IP**：全球唯一，可以从任何地方访问（但通常要收费）

## 六、常见错误与原理

| 错误信息 | 根本原因 | 原理 |
|---------|---------|------|
| `dhclient` 卡死 | 网络没有 DHCP 服务器 | 广播没人回应 |
| `Name or service not known` | DNS 没配 | 不知道域名对应的 IP |
| `ping x.x.x.x` 100% 丢包 | 网关没配或配置错误 | 数据包不知道往哪送 |
| `Address already assigned` | IP 重复分配 | 本机已有这个 IP |
| `RTNETLINK: File exists` | 网关已存在 | 不能重复添加 |

## 七、命令速查

```bash
# 查看 IP 配置
ip addr show enp1s0

# 配置静态 IP
sudo ip addr add 10.3.36.213/24 dev enp1s0

# 查看网关
ip route show

# 添加默认网关
sudo ip route add default via 10.3.36.254 dev enp1s0

# 配置 DNS
echo "nameserver 223.5.5.5" | sudo tee /etc/resolv.conf
echo "nameserver 114.114.114.114" | sudo tee -a /etc/resolv.conf

# 排障测试
ping 10.3.36.254        # 内网
ping 223.5.5.5          # 外网
ping baidu.com          # DNS
```

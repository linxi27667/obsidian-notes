# Linux 有线网络无法上网：排障与网络基础知识

---

## 一、这次遇到了什么故障？

> [!NOTE] 故障现象
> 1. `dhclient` 自动获取 IP 卡死无响应
> 2. 手动配置 IP 后，无法 ping 通百度
> 3. 域名 ping 报 `Name or service not known`，公网 IP 100% 丢包
>
> **最终原因**：不是硬件坏了，而是少了 ==网关== 和 ==DNS== 配置

---

## 二、网络能通的三层楼（为什么需要这么多步骤）

上网就像寄快递，整个过程分为**三层楼**：

```
            ┌─────────────────────────┐
   第三层    │  域名解析 (DNS)           │  你说 "baidu.com"
            │  DNS 服务器 223.5.5.5     │  它告诉你 IP 是 39.156.66.10
            └─────────────────────────┘
                      ↓
            ┌─────────────────────────┐
   第二层    │  公网出口 (默认网关)        │  所有不是本地的包
            │  网关 10.3.36.254        │  都交给它出去
            └─────────────────────────┘
                      ↓
            ┌─────────────────────────┐
   第一层    │  本机 IP 地址             │  你的电脑的门牌号
            │  10.3.36.213/24         │  必须和局域网在同一段
            └─────────────────────────┘
```

> [!TIP] 为什么三层缺一不可
> - 没配 IP → 快递没有寄件人地址，快递柜不收
> - 没配网关 → 快递不知道往哪个方向寄，出不了门
> - 没配 DNS → 你只知道对方姓名，不知道手机号，无法联系
>
> **任何一层缺失，网络就不通**

---

## 三、每个步骤对应的网络知识

### 第一步：查物理连接（`ethtool`）

**学到了什么？**
- 有线网卡靠 ==网线== 物理连接交换机，不是蓝牙/WiFi
- `Link detected: yes` = 网线插好了、网口没坏、交换机没关机
- 排除的是**物理层（Physical Layer）**的故障

> [!NOTE] OSI 七层参考
> 1. 物理层（Physical）：网线、光纤、网口电信号
> 2. 数据链路层（Data Link）：MAC 地址、交换机学习
> 3. **网络层（Network）：IP 地址、路由、网关** ← 我们重点在这里
> 4. 传输层（Transport）：TCP/UDP 端口
> 5. 应用层（Application）：HTTP、DNS、MQTT

---

### 第二步：放弃 DHCP（`dhclient` 卡死）

**学到了什么？**
- 网络设备获取 IP 有两种方式：
  - **DHCP**（自动）：路由器/服务器自动分配，你什么都不用管
  - **静态 IP**（手动）：自己指定 IP 地址
- `dhclient` 卡死 = 网络里**没有 DHCP 服务器**，无法自动获取
- 此时只能**手动指定静态 IP**，绕过 DHCP

> [!NOTE] DHCP 是什么
> DHCP = Dynamic Host Configuration Protocol
> 想象成：你去酒店前台登记，前台自动给你分配一个房间号，不用你自己选
> 企业/办公室网络通常有 DHCP 服务器（路由器内置）
> 学校实验室/工控机等特殊网络可能没有 DHCP，必须手动配

---

### 第三步：配置本机 IP（`ip addr add`）

**学到了什么？**
- IP 地址 = 网络上的"门牌号"，用来唯一标识你的设备
- `10.3.36.213/24` 是什么意思？
  - `10.3.36.213` = IP 地址
  - `/24` = 子网掩码是 255.255.255.0，表示前 24 位是网络号
  - 同在一个 10.3.36.x 网段的设备可以**直接互访**
- 配置静态 IP 之前，需要知道**局域网的网络号**，否则会冲突

> [!TIP] 为什么 IP 不能随便写
> 比如你在 10.3.36.x 网段，配了一个 192.168.1.x 的 IP
> 那你就收不到同局域网其他设备的包，因为它们不在一个"街道"上

---

### 第四步：配置默认网关（`ip route add default`）

**学到了什么？**

> [!NOTE] 什么是网关（Gateway）
> 网关 = 网络的"大门"，所有不是本地网段的包都从这出去
>
> 想象：你家小区（局域网），你想寄信到外省（公网）
> 你把信交给小区门卫（网关），门卫帮你转发出去
> 没有网关，你连小区都出不去

- 本地网络（10.3.36.x）内的设备可以**直接互访**（交换机负责）
- 访问外地网络（baidu.com）必须经过**网关**转发
- `10.3.36.254` 就是这个网络的网关地址（通常路由器是 .1 或 .254）
- 没有配置网关 = **只能 ping 通局域网内的设备，无法访问外网**

> [!WARNING] ping 公网 IP 和 ping 域名的区别
> - `ping 223.5.5.5`（公网 IP）→ 不需要 DNS，直接找 IP
> - `ping baidu.com`（域名）→ 需要 DNS 先把域名转成 IP
>
> 如果 ping IP 通但 ping 域名不通 → 问题在 DNS
> 如果 ping IP 也不通 → 问题在网关

---

### 第五步：配置 DNS（`/etc/resolv.conf`）

**学到了什么？**

> [!NOTE] 什么是 DNS
> DNS = Domain Name System，域名系统
> 相当于网络的"电话簿"
>
> 你说 "baidu.com" → DNS 告诉你 "39.156.66.10"
> 你说 "google.com" → DNS 告诉你 "142.250.191.78"
>
> 没有 DNS，你只能记住 IP 才能访问网站（没人能做到）

- 常用公共 DNS：
  - `223.5.5.5`（阿里 DNS）
  - `114.114.114.114`（114 DNS）
  - `8.8.8.8`（Google DNS）
- `/etc/resolv.conf` = Linux 操作系统记录 DNS 服务器地址的文件
- DNS 配置错误 → 能 ping IP，但无法 ping 域名

> [!TIP] DNS 故障的特征
> `Name or service not known` = DNS 解析失败
> 此时 ping IP 地址（如 ping 223.5.5.5）应该是通的
> 如果 ping IP 也不通 → 不是 DNS 的问题，是网关还没通

---

### 第六步：分层验证网络

**学到了什么？排障要从内到外，一层层排查：**

| 步骤 | 命令 | 验证内容 | 如果失败说明 |
|------|------|----------|-------------|
| 1 | `sudo ethtool enp1s0` | 物理层正常 | 网线没插好/网口坏了 |
| 2 | `ping 10.3.36.254` | 内网（网关）通了 | 网关 IP 配错了 |
| 3 | `ping 223.5.5.5` | 外网通了 | 网关没配/配置错误 |
| 4 | `ping baidu.com` | DNS 解析正常 | DNS 没配/配置错误 |

> [!NOTE] 黄金排障顺序
> 先排除物理层 → 再测内网 → 再测外网 → 最后测 DNS
> 这样做的好处：每次只排查一个问题，不会一团乱

## 四、命令与网络知识对照表

| 命令 | 对应网络知识 | 说明 |
|------|------------|------|
| `ip addr show` | IP 地址 | 查看本机 IP 和状态 |
| `ip addr add x.x.x.x/24 dev eth0` | 静态 IP 配置 | 手动指定 IP，绕过 DHCP |
| `ip route show` | 路由表/网关 | 查看数据包该往哪走 |
| `ip route add default via x.x.x.x` | 默认网关 | 所有不明目的地的包都发到这里 |
| `echo "nameserver x.x.x.x" \| tee /etc/resolv.conf` | DNS 服务器 | 指定域名解析服务器 |
| `ping x.x.x.x` | 连通性测试 | 测试到某个 IP 是否可达 |
| `ping domain.com` | DNS 解析测试 | 测试域名是否能解析 |
| `ethtool eth0` | 物理层状态 | 查看网线是否插好、速率多少 |

## 五、高频报错与原理

| 报错 | 根本原因 | 原理 |
|------|---------|------|
| `dhclient` 卡死 | 没有 DHCP 服务器 | 客户端发广播请求，没人回应就卡住 |
| `ping: Name or service not known` | DNS 未配置 | 系统不知道域名对应的 IP |
| `ping x.x.x.x` 100% 丢包 | 网关未配置或错误 | 数据包找不到出口，被丢弃 |
| `RTNETLINK: File exists` | 网关已存在 | 不要重复添加，先删再加 |
| `Address already assigned` | IP 重复 | 本机已有这个 IP，不能重复配 |

## 六、一键速查命令

```bash
# 1. 查看网卡信息
ip addr show enp1s0

# 2. 查看物理连接
sudo ethtool enp1s0

# 3. 配置静态 IP
sudo ip addr add 10.3.36.213/24 dev enp1s0

# 4. 配置网关（删旧+加新）
sudo ip route del default && sudo ip route add default via 10.3.36.254 dev enp1s0

# 5. 配置 DNS
echo "nameserver 223.5.5.5" | sudo tee /etc/resolv.conf
echo "nameserver 114.114.114.114" | sudo tee -a /etc/resolv.conf

# 6. 分层测试
ping 10.3.36.254 && ping 223.5.5.5 && ping baidu.com
```

## 七、核心结论

> [!TIP] Linux 有线网上网的三大必备条件
> 1. **静态 IP** 配置正确（和局域网同一网段）
> 2. **默认网关** 配置正确（数据包出局的出口）
> 3. **DNS 服务器** 配置正确（域名 → IP 转换）

> [!NOTE] 排障黄金逻辑
> **先查物理层（ethtool）→ ping 网关（内网）→ ping 公网 IP（外网）→ ping 域名（DNS）**

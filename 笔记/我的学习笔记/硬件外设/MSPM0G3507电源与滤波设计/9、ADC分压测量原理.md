# ADC 分压测量原理 （“一箭双雕”设计，板载下拉电阻的妙用）

## 核心概念

- ==ADC 引脚是高阻抗电压表== - 测量的是电压，不是电流；流入引脚的电流几乎为 0
- ==内部 R8 (100kΩ) 是“固定秤砣”== - 板载下拉电阻，与外部元件串联分压，将阻值变化转化为电压变化
- ==谁阻值大，谁抢的电压多== - 3.3V 总电压由外部元件和 R8 瓜分，阻值占比决定分得电压
- ==一箭双雕设计== - 同一个 PA2 接口，既能测外部电压，又能接传感器测温度/光照

---

## 一、分压核心原理

### 1.1 为什么 PA2 量的是 R8 两端的电压？

**物理连接位置**：PA2 引脚**物理上就接在 R8 的头顶上**。

```
外部信号 (3.3V 或传感器)
        │
        ▼
      [PA2 排针] ◄── 单片机引脚接在这里
        │
        │  ← 这个点就是 R8 的"上端"
       ┌┴┐
       │ │ R8 (100kΩ)
       └┬┘
        │
        │  ← 这个点就是 R8 的"下端"
       GND (0V)
```

**电压的本质 = 两点之间的高度差**：
- R8 下端接 GND = 0V（地面）
- R8 上端接 PA2 = 某个电压值
- 所以 **PA2 对地的电压 = R8 两端的电压**

**水管分水比喻**：
把电流想象成水流：
- 3.3V 是水箱（高处）
- R8 是一段细水管
- GND 是下水道（低处）
- PA2 就像**水压表**，装在这段细水管的**入口**，量的就是这段水管两头的压力差

### 1.2 电路结构

```
外部信号源/传感器 → PA2 排针 → 内部 R8 (100kΩ) → GND
                         ↓
                      单片机 ADC 采样点
```

### 1.3 分压公式

$$V_{PA2} = V_{source} \times \frac{R8}{R_{ext} + R8}$$

- $V_{PA2}$：单片机引脚测到的电压
- $V_{source}$：外部信号源电压（如 3.3V 或待测电池电压）
- $R8$：内部下拉电阻（固定 100kΩ）
- $R_{ext}$：外部串联电阻或传感器阻值

### 1.4 为什么 ADC 不看电流？

- 单片机 ADC 引脚内部阻抗极大（相当于断路）
- 外部电流几乎 100% 流过 R8 到地，不流入单片机
- 引脚测量的是电流在 R8 上产生的**压降**（$U = I \times R8$）

## 二、两种典型应用场景

### 2.1 场景 A：测量外部高电压（如 5V 电池）

**目的**：保护引脚不超过 3.3V，同时测量外部电压。

**接法**：
- 外部串联一个电阻（如 100kΩ）在 5V 电源和 PA2 排针之间。
- 内部 R8 (100kΩ) 接地。

**计算**：
$$V_{PA2} = 5V \times \frac{100k}{100k + 100k} = 2.5V$$
- 2.5V 在安全范围内（< 3.3V）
- 代码中读取 2.5V，乘以 2 还原为 5V

### 2.2 场景 B：测量电阻型传感器（如 NTC 热敏电阻）

**目的**：将传感器阻值变化转化为电压变化。

**接法**：
- 传感器（如 NTC 100kΩ）一端接 3.3V，一端接 PA2 排针。
- 内部 R8 (100kΩ) 接地。

**计算**：
- **温度升高** → NTC 阻值减小（如 50kΩ）
  $$V_{PA2} = 3.3V \times \frac{100k}{50k + 100k} = 2.2V$$
- **温度降低** → NTC 阻值增大（如 200kΩ）
  $$V_{PA2} = 3.3V \times \frac{100k}{200k + 100k} = 1.1V$$

**结论**：温度越高，PA2 电压越高；温度越低，PA2 电压越低。

## 三、为什么内部电阻选 100kΩ？

| 原因 | 说明 |
|------|------|
| **匹配常见传感器** | 市面最常用 NTC 温度计在 25℃ 时阻值刚好是 100kΩ |
| **降低功耗** | 高阻值意味着回路电流极小（3.3V / 200kΩ ≈ 16.5μA），省电 |
| **高阻抗匹配** | ADC 输入阻抗极高，100kΩ 分压电阻不会造成明显负载效应 |

## 四、软件配置极简指南

在 TI SysConfig 中只需三步：

1. **开启 ADC0 模块**。
2. **配置通道**：选择 PA2 引脚对应的 ADC 通道。
3. **选择参考电压**：选内部 2.5V 基准（Internal VREF 2.5V）。

**代码读取**：
- 调用读取函数（如 `DL_ADC12_getMemResult`）
- 返回 0 ~ 4095 之间的数字（12 位 ADC 分辨率）
- 换算公式：$Voltage = \frac{ADC\_Value}{4095} \times 2.5V$

## 五、常见误区纠正

### 5.1 误区：电阻变低，输入引脚电流变大，所以读数变大

**纠正**：
- 电阻变低 → 回路总电流确实变大
- 但 ADC 不看电流，看的是**电压**
- 电流变大 → 在固定 R8 上产生的压降变大 → 引脚电压变高 → 读数变大
- **本质是电压变化，不是电流直接流入引脚**

### 5.2 误区：ADC 引脚会吸收电流

**纠正**：
- ADC 引脚阻抗极大，几乎不吸收电流
- 电流全部流过 R8 到地
- 引脚只是“旁听”R8 两端的电压

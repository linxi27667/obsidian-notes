# FreeRTOS 信号量与互斥量 （用于任务同步、资源共享和优先级继承）

> 创建时间: 2026-05-06

## 核心概念

### 二值信号量 vs 计数信号量 vs 互斥量

| 类型 | 取值范围 | 主要用途 | 是否支持优先级继承 |
|------|----------|----------|-------------------|
| 二值信号量 | 0或1 | 任务同步、事件标志 | 否 |
| 计数信号量 | 0 ~ 最大值 | 资源计数、限流 | 否 |
| 互斥量 | 0或1 | 临界区保护、资源共享 | 是 |
| 递归互斥量 | 递归计数 | 同一任务多次加锁 | 是 |

### 本质区别
- ==信号量是"令牌"== - 产生事件的地方Give，消费事件的地方Take
- ==互斥量是"钥匙"== - 加锁方Take，解锁方Give，强调所有权
- ==互斥量支持优先级继承== - 解决优先级反转问题

## 一、二值信号量

### 1.1 概述
- 最简单的同步机制，状态只有"有"（1）和"无"（0）
- 用于中断与任务同步、任务间事件通知

### 1.2 创建与操作

```c
// 创建二值信号量
SemaphoreHandle_t xSemaphoreCreateBinary(void);

// 获取信号量（Take）
BaseType_t xSemaphoreTake(
    SemaphoreHandle_t xSemaphore,
    TickType_t xTicksToWait
);

// 释放信号量（Give）
BaseType_t xSemaphoreGive(SemaphoreHandle_t xSemaphore);
```

| 函数 | 参数 | 类型 | 说明 |
|------|------|------|------|
| `xSemaphoreTake` | `xSemaphore` | `SemaphoreHandle_t` | 信号量句柄 |
| | `xTicksToWait` | `TickType_t` | 阻塞等待时间（tick数，`portMAX_DELAY`=无限等待） |
| `xSemaphoreGive` | `xSemaphore` | `SemaphoreHandle_t` | 信号量句柄 |

**返回值**: `xSemaphoreTake` 返回 `pdPASS`（获取成功）或 `pdFAIL`（超时）；`xSemaphoreGive` 返回 `pdPASS` 或 `errQUEUE_FULL`

### 1.3 中断与任务同步示例

```c
static SemaphoreHandle_t xBinarySem;

// 中断服务例程
void EXTI_IRQHandler(void)
{
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;

    // 中断中释放信号量
    xSemaphoreGiveFromISR(xBinarySem, &xHigherPriorityTaskWoken);
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}

// 同步等待任务
void vSyncTask(void *pvParameters)
{
    while (1) {
        // 阻塞等待信号量
        xSemaphoreTake(xBinarySem, portMAX_DELAY);
        // 收到信号量后处理
        handle_interrupt_event();
    }
}

void main(void)
{
    xBinarySem = xSemaphoreCreateBinary();
    // 注意：二值信号量创建时初始为空（0）
    // 如果需要初始为可用状态，需先Give一次

    xTaskCreate(vSyncTask, "SyncTask", 256, NULL, 1, NULL);
    vTaskStartScheduler();
}
```

## 二、计数信号量

### 2.1 概述
- 可以计数多个资源/事件
- 用于资源管理、生产者-消费者限流

### 2.2 创建与操作

```c
SemaphoreHandle_t xSemaphoreCreateCounting(
    UBaseType_t uxMaxCount,
    UBaseType_t uxInitialCount
);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `uxMaxCount` | `UBaseType_t` | 最大计数值 |
| `uxInitialCount` | `UBaseType_t` | 初始计数值 |

**返回值**: `SemaphoreHandle_t` — 成功返回信号量句柄，失败返回`NULL`

### 2.3 资源管理示例

```c
static SemaphoreHandle_t xResSem;
#define MAX_RESOURCES 5

void main(void)
{
    // 创建计数信号量，初始5个资源都可用
    xResSem = xSemaphoreCreateCounting(MAX_RESOURCES, MAX_RESOURCES);

    xTaskCreate(vResourceUser, "User1", 256, NULL, 1, NULL);
    xTaskCreate(vResourceUser, "User2", 256, NULL, 1, NULL);

    vTaskStartScheduler();
}

void vResourceUser(void *pvParameters)
{
    while (1) {
        // 获取资源（计数减1）
        if (xSemaphoreTake(xResSem, portMAX_DELAY) == pdPASS) {
            // 使用资源
            use_resource();
            // 释放资源（计数加1）
            xSemaphoreGive(xResSem);
        }
    }
}
```

## 三、互斥量

### 3.1 概述
- ==核心特性：优先级继承== - 防止优先级反转
- 用于保护共享资源的临界区
- 具有"所有权"概念：谁Take必须谁Give

### 3.2 创建与操作

```c
// 创建互斥量
SemaphoreHandle_t xSemaphoreCreateMutex(void);

// 创建递归互斥量
SemaphoreHandle_t xSemaphoreCreateRecursiveMutex(void);

// 操作（与信号量相同）
xSemaphoreTake(xMutex, portMAX_DELAY);
    // 临界区
xSemaphoreGive(xMutex);
```

### 3.3 优先级反转问题

**场景**：
1. 低优先级任务L获取互斥量，进入临界区
2. 高优先级任务H抢占L，也需要该互斥量，进入阻塞
3. 中优先级任务M就绪，抢占L执行
4. 结果：H被M间接阻塞 — **优先级反转**

**解决方案**：优先级继承
- 当H等待L持有的互斥量时，L临时继承H的优先级
- L退出临界区后，恢复原优先级
- H立即获得CPU

### 3.4 共享资源保护示例

```c
static SemaphoreHandle_t xSPI_Mutex;

// 任务A：使用SPI
void vTaskA_SPI(void *pvParameters)
{
    while (1) {
        // 获取SPI总线使用权
        xSemaphoreTake(xSPI_Mutex, portMAX_DELAY);

        // 使用SPI设备（临界区）
        spi_select_device(DEVICE_A);
        spi_transfer(data, len);
        spi_deselect_device(DEVICE_A);

        // 释放SPI总线
        xSemaphoreGive(xSPI_Mutex);

        vTaskDelay(pdMS_TO_TICKS(500));
    }
}

// 任务B：也使用SPI
void vTaskB_SPI(void *pvParameters)
{
    while (1) {
        xSemaphoreTake(xSPI_Mutex, portMAX_DELAY);

        spi_select_device(DEVICE_B);
        spi_transfer(data, len);
        spi_deselect_device(DEVICE_B);

        xSemaphoreGive(xSPI_Mutex);

        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}
```

### 3.5 递归互斥量

```c
void vComplexFunction(void)
{
    // 同一任务可以多次Take
    xSemaphoreTakeRecursive(xRecMutex, portMAX_DELAY);
        // 第一层临界区
        vNestedFunction();  // 内部也调用Take
    xSemaphoreGiveRecursive(xRecMutex);
}

void vNestedFunction(void)
{
    xSemaphoreTakeRecursive(xRecMutex, portMAX_DELAY);
        // 嵌套临界区
    xSemaphoreGiveRecursive(xRecMutex);
}
```

## 四、中断安全的信号量操作

```c
// 中断中释放信号量
BaseType_t xSemaphoreGiveFromISR(
    SemaphoreHandle_t xSemaphore,
    BaseType_t *pxHigherPriorityTaskWoken
);

// 中断中获取信号量（不常用）
BaseType_t xSemaphoreTakeFromISR(
    SemaphoreHandle_t xSemaphore,
    BaseType_t *pxHigherPriorityTaskWoken
);
```

| 函数 | 参数 | 类型 | 说明 |
|------|------|------|------|
| `xSemaphoreGiveFromISR` | `xSemaphore` | `SemaphoreHandle_t` | 信号量句柄 |
| | `pxHigherPriorityTaskWoken` | `BaseType_t *` | 输出参数，若因释放导致高优先级任务就绪则设为`pdTRUE` |
| `xSemaphoreTakeFromISR` | `xSemaphore` | `SemaphoreHandle_t` | 信号量句柄 |
| | `pxHigherPriorityTaskWoken` | `BaseType_t *` | 输出参数，若因获取导致高优先级任务就绪则设为`pdTRUE` |

## 五、完整示例

### 示例：UART发送同步

```c
static SemaphoreHandle_t xUartSem;
static QueueHandle_t xUartQueue;

// UART发送任务
void vUartTxTask(void *pvParameters)
{
    UartMsg_t msg;

    while (1) {
        // 等待队列中的消息
        if (xQueueReceive(xUartQueue, &msg, portMAX_DELAY) == pdPASS) {
            // 启动DMA传输
            uart_start_dma(msg.data, msg.len);

            // 等待传输完成信号量
            xSemaphoreTake(xUartSem, portMAX_DELAY);

            // 传输完成，发送通知
            notify_tx_complete();
        }
    }
}

// DMA完成中断
void DMA_IRQHandler(void)
{
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;

    if (DMA_GetITStatus(DMA_FLAG_TC)) {
        // 释放信号量通知任务
        xSemaphoreGiveFromISR(xUartSem, &xHigherPriorityTaskWoken);
        DMA_ClearITPendingBit(DMA_FLAG_TC);
    }

    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}
```

# FreeRTOS 事件组与任务通知 （多事件同步和轻量级任务通信机制）

> 创建时间: 2026-05-06

## 核心概念

### 事件组
- ==多事件同步机制== - 一个事件组可表示24个独立事件位
- ==等待多个条件== - 任务可等待"所有位设置"或"任一位设置"
- ==自动/手动清除== - 可选择等待成功后自动清除位或保持

### 任务通知
- ==轻量级通信== - 直接发送到目标任务，不需要创建额外对象
- ==内置于TCB== - 每个任务自带通知值和通知状态
- ==四种模式== - 覆盖值、设位、增加、发送完整通知值
- ==性能最高== - 比队列/信号量更快，内存占用更小

## 一、事件组

### 1.1 基本概念

事件组本质是一组24个标志位（Event Bits），每位代表一个事件。

```
Event Group: 0x000000 ~ 0xFFFFFF
              │││...│││
              ││   ││└─ Bit 0: 事件0
              ││   │└── Bit 1: 事件1
              ││    └── Bit 2: 事件2
              └└...───── Bit 23: 事件23
```

### 1.2 创建与操作

```c
// 创建事件组
EventGroupHandle_t xEventGroupCreate(void);

// 设置事件位（任务级）
EventBits_t xEventGroupSetBits(
    EventGroupHandle_t xEventGroup,
    const EventBits_t uxBitsToSet
);

// 等待事件位
EventBits_t xEventGroupWaitBits(
    EventGroupHandle_t xEventGroup,
    const EventBits_t uxBitsToWaitFor,
    const BaseType_t xClearOnExit,
    const BaseType_t xWaitForAllBits,
    TickType_t xTicksToWait
);
```

`xEventGroupSetBits` 参数表：

| 参数 | 类型 | 说明 |
|------|------|------|
| `xEventGroup` | `EventGroupHandle_t` | 事件组句柄 |
| `uxBitsToSet` | `const EventBits_t` | 要设置的位（使用`|`运算组合多个位） |

**返回值**: 更新前的位值

`xEventGroupWaitBits` 参数表：

| 参数 | 类型 | 说明 |
|------|------|------|
| `xEventGroup` | `EventGroupHandle_t` | 事件组句柄 |
| `uxBitsToWaitFor` | `const EventBits_t` | 等待哪些位（使用`|`运算组合） |
| `xClearOnExit` | `const BaseType_t` | `pdTRUE`=成功后清除位，`pdFALSE`=保持 |
| `xWaitForAllBits` | `const BaseType_t` | `pdTRUE`=等待所有位，`pdFALSE`=任一位即可 |
| `xTicksToWait` | `TickType_t` | 阻塞等待时间（tick数） |

**返回值**: 触发等待时的位值，超时返回`0`

### 1.3 位操作详解

```c
// 位常量定义
#define WIFI_CONNECTED_BIT    (1 << 0)   // Bit 0
#define MQTT_READY_BIT        (1 << 1)   // Bit 1
#define SENSOR_INIT_BIT       (1 << 2)   // Bit 2
#define CONFIG_LOADED_BIT     (1 << 3)   // Bit 3

// 设置多个位
xEventGroupSetBits(xEventGroup, WIFI_CONNECTED_BIT | MQTT_READY_BIT);

// 清除位（通过 xEventGroupWaitBits 的 xClearOnExit 参数）
// 或主动清除
xEventGroupClearBits(xEventGroup, WIFI_CONNECTED_BIT);
```

### 1.4 使用场景

**场景1：等待多个条件就绪**
```c
static EventGroupHandle_t xEventGroup;

// 初始化任务
void vInitTask(void *pvParameters)
{
    // 初始化WiFi
    wifi_init();
    xEventGroupSetBits(xEventGroup, WIFI_CONNECTED_BIT);

    // 初始化MQTT
    mqtt_init();
    xEventGroupSetBits(xEventGroup, MQTT_READY_BIT);

    // 初始化传感器
    sensor_init();
    xEventGroupSetBits(xEventGroup, SENSOR_INIT_BIT);
}

// 主应用任务
void vAppTask(void *pvParameters)
{
    // 等待所有初始化完成
    EventBits_t bits = xEventGroupWaitBits(
        xEventGroup,
        WIFI_CONNECTED_BIT | MQTT_READY_BIT | SENSOR_INIT_BIT,
        pdFALSE,           // 不清除位
        pdTRUE,            // 等待所有位
        portMAX_DELAY      // 无限等待
    );

    // 所有初始化完成，开始主循环
    while (1) {
        main_loop();
    }
}
```

**场景2：任一事件触发即处理**
```c
void vMonitorTask(void *pvParameters)
{
    while (1) {
        EventBits_t bits = xEventGroupWaitBits(
            xEventGroup,
            OVERTEMP_BIT | LOW_BATTERY_BIT | DOOR_OPEN_BIT,
            pdTRUE,            // 清除已处理的位
            pdFALSE,           // 任一位即可
            portMAX_DELAY
        );

        if (bits & OVERTEMP_BIT) {
            handle_overtemp();
        }
        if (bits & LOW_BATTERY_BIT) {
            handle_low_battery();
        }
        if (bits & DOOR_OPEN_BIT) {
            handle_door_open();
        }
    }
}
```

### 1.5 中断安全API

```c
// 中断中设置事件位
BaseType_t xEventGroupSetBitsFromISR(
    EventGroupHandle_t xEventGroup,
    const EventBits_t uxBitsToSet,
    BaseType_t *pxHigherPriorityTaskWoken
);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `xEventGroup` | `EventGroupHandle_t` | 事件组句柄 |
| `uxBitsToSet` | `const EventBits_t` | 要设置的位（使用`|`运算组合） |
| `pxHigherPriorityTaskWoken` | `BaseType_t *` | 输出参数，若导致高优先级任务就绪则设为`pdTRUE` |

**返回值**: `BaseType_t` — `pdPASS`表示命令成功发送到Timer任务，`pdFAIL`表示命令队列满

## 二、任务通知

### 2.1 概述

每个FreeRTOS任务拥有一个32位通知值和一个通知状态，可直接被其他任务或中断修改。

```
Task Control Block (TCB)
├── ...
├── ulNotifiedValue      // 32位通知值
├── ucNotifyState        // 通知状态 (pending/not-pending)
└── ...
```

### 2.2 四种通知模式

| 模式 | 行为 | 适用场景 |
|------|------|----------|
| `eSetValueWithOverwrite` | 覆盖当前值 | 最新状态更新 |
| `eSetValueWithoutOverwrite` | 仅当无待处理通知时设置 | 不丢失待处理通知 |
| `eSetBits` | 按位OR设置 | 事件标志（类似事件组） |
| `eIncrement` | 通知值+1 | 计数信号量替代 |
| `eNoAction` | 仅发送通知，不改变值 | 唤醒任务 |

### 2.3 发送通知

```c
// 任务级发送
BaseType_t xTaskGenericNotify(
    TaskHandle_t xTaskToNotify,
    uint32_t ulValue,
    eNotifyAction eAction,
    uint32_t *pulPreviousNotifyValue
);

// 简化版本
BaseType_t xTaskNotifyGive(TaskHandle_t xTaskToNotify);
uint32_t ulTaskNotifyValueClear(TaskHandle_t xTask, uint32_t ulBitsToClear);
```

`xTaskGenericNotify` 参数表：

| 参数 | 类型 | 说明 |
|------|------|------|
| `xTaskToNotify` | `TaskHandle_t` | 目标任务句柄 |
| `ulValue` | `uint32_t` | 通知值（根据`eAction`模式有不同行为） |
| `eAction` | `eNotifyAction` | 通知模式（见上方四种模式表） |
| `pulPreviousNotifyValue` | `uint32_t *` | 可选：输出参数，获取修改前的通知值（可设为`NULL`） |

**返回值**: `BaseType_t` — `pdPASS`表示通知成功，`pdFAIL`表示`eSetValueWithoutOverwrite`模式下已有待处理通知

### 2.4 接收通知

```c
// 等待通知
BaseType_t xTaskNotifyWait(
    uint32_t ulBitsToClearOnEntry,
    uint32_t ulBitsToClearOnExit,
    uint32_t *pulNotificationValue,
    TickType_t xTicksToWait
);

// 简化版本：获取二值信号量式通知
uint32_t ulTaskNotifyTake(
    BaseType_t xClearCountOnExit,
    TickType_t xTicksToWait
);
```

`xTaskNotifyWait` 参数表：

| 参数 | 类型 | 说明 |
|------|------|------|
| `ulBitsToClearOnEntry` | `uint32_t` | 进入函数时清除的位（`0xFFFFFFFF`清除全部） |
| `ulBitsToClearOnExit` | `uint32_t` | 成功退出时清除的位 |
| `pulNotificationValue` | `uint32_t *` | 输出参数，接收通知值（可设为`NULL`） |
| `xTicksToWait` | `TickType_t` | 阻塞等待时间 |

`ulTaskNotifyTake` 参数表：

| 参数 | 类型 | 说明 |
|------|------|------|
| `xClearCountOnExit` | `BaseType_t` | `pdTRUE`=通知值清零，`pdFALSE`=通知值减1 |
| `xTicksToWait` | `TickType_t` | 阻塞等待时间 |

## 三、完整示例

### 示例1：任务通知替代二值信号量（更高效）

```c
static TaskHandle_t xHandlerTaskHandle;

// 使用任务通知替代二值信号量
void EXTI_IRQHandler(void)
{
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;

    // 直接通知任务，无需创建信号量
    vTaskNotifyGiveFromISR(xHandlerTaskHandle, &xHigherPriorityTaskWoken);
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}

void vHandlerTask(void *pvParameters)
{
    while (1) {
        // 等待通知（替代xSemaphoreTake）
        ulTaskNotifyTake(pdTRUE, portMAX_DELAY);
        handle_event();
    }
}

void main(void)
{
    xTaskCreate(vHandlerTask, "Handler", 256, NULL, 1, &xHandlerTaskHandle);
    vTaskStartScheduler();
}
```

### 示例2：任务通知替代计数信号量

```c
static TaskHandle_t xWorkerTaskHandle;

// 多个中断源增加计数
void IRQ_Handler_1(void)
{
    BaseType_t xYield = pdFALSE;
    vTaskNotifyGiveFromISR(xWorkerTaskHandle, &xYield);
    portYIELD_FROM_ISR(xYield);
}

void IRQ_Handler_2(void)
{
    BaseType_t xYield = pdFALSE;
    vTaskNotifyGiveFromISR(xWorkerTaskHandle, &xYield);
    portYIELD_FROM_ISR(xYield);
}

// 工作处理任务
void vWorkerTask(void *pvParameters)
{
    uint32_t count;
    while (1) {
        count = ulTaskNotifyTake(pdFALSE, portMAX_DELAY);  // pdFALSE = 减1
        // count表示有多少个事件待处理
        while (count--) {
            process_one_item();
        }
    }
}
```

### 示例3：任务通知传递数据

```c
static TaskHandle_t xSensorTaskHandle;

// 传感器任务发送数据给处理任务
void vSensorTask(void *pvParameters)
{
    while (1) {
        uint32_t sensor_value = read_adc();

        // 发送数据值给处理任务
        xTaskNotify(xProcessingTaskHandle, sensor_value, eSetValueWithOverwrite);

        vTaskDelay(pdMS_TO_TICKS(100));
    }
}

// 处理任务接收数据
void vProcessingTask(void *pvParameters)
{
    uint32_t value;
    while (1) {
        xTaskNotifyWait(0, 0xFFFFFFFF, &value, portMAX_DELAY);
        process_sensor_value(value);
    }
}
```

## 四、通信机制对比

| 机制 | 内存占用 | 速度 | 方向 | 数据类型 | 适用场景 |
|------|----------|------|------|----------|----------|
| 队列 | 需创建对象 | 中等 | 多对多 | 任意大小 | 数据传输 |
| 二值信号量 | 需创建对象 | 中等 | 多对多 | 无数据 | 任务同步 |
| 计数信号量 | 需创建对象 | 中等 | 多对多 | 无数据 | 资源计数 |
| 互斥量 | 需创建对象 | 中等 | 一对多 | 无数据 | 临界区保护 |
| 事件组 | 需创建对象 | 快 | 多对多 | 24位标志 | 多事件同步 |
| **任务通知** | **内置于TCB** | **最快** | **一对一对一** | **32位值** | **轻量通信** |

# FreeRTOS 中断管理与低功耗 （中断安全API规则和Tickless idle模式）

> 创建时间: 2026-05-06

## 核心概念

### 中断管理
- ==ISR专用API== - 中断中必须使用`FromISR`后缀的函数
- ==临界区保护== - 中断中使用 `taskENTER_CRITICAL_FROM_ISR()` / `taskEXIT_CRITICAL_FROM_ISR()`
- ==上下文切换请求== - 通过 `portYIELD_FROM_ISR()` 请求在ISR结束后切换任务
- ==中断优先级配置== - 需确保FreeRTOS管理的优先级高于中断

### Tickless Idle模式
- ==低功耗核心机制== - 空闲任务运行时关闭tick中断以省电
- ==自动唤醒== - 在下一个tick到期前自动唤醒
- ==适合电池供电设备== - 显著降低空闲功耗

## 一、中断安全API规则

### 1.1 FromISR 后缀函数

| 普通API | ISR安全API |
|---------|-----------|
| `xQueueSend()` | `xQueueSendFromISR()` |
| `xQueueReceive()` | `xQueueReceiveFromISR()` |
| `xSemaphoreTake()` | `xSemaphoreTakeFromISR()` |
| `xSemaphoreGive()` | `xSemaphoreGiveFromISR()` |
| `xEventGroupSetBits()` | `xEventGroupSetBitsFromISR()` |
| `vTaskDelay()` | ❌ 不可在ISR中使用 |
| `xTaskNotify()` | `xTaskNotifyFromISR()` |

### 1.2 为什么需要区分

- 中断上下文不能阻塞
- 任务切换需要延迟到ISR结束后执行
- `FromISR`函数内部使用不同的同步机制，不依赖调度器

### 1.3 上下文切换机制

```c
void Some_IRQHandler(void)
{
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;

    // 发送数据到队列
    xQueueSendFromISR(xQueue, &data, &xHigherPriorityTaskWoken);

    // 如果有更高优先级任务被唤醒，请求上下文切换
    // 实际切换发生在ISR退出后
    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}
```

## 二、临界区管理

### 2.1 任务中的临界区

```c
void vTask(void *pvParameters)
{
    while (1) {
        taskENTER_CRITICAL();
            // 临界区代码
            // 禁止所有中断
            shared_counter++;
        taskEXIT_CRITICAL();

        // 正常工作
    }
}
```

### 2.2 中断中的临界区

```c
void Some_IRQHandler(void)
{
    UBaseType_t uxSavedInterruptStatus;

    uxSavedInterruptStatus = taskENTER_CRITICAL_FROM_ISR();
        // 中断中的临界区
        critical_data_update();
    taskEXIT_CRITICAL_FROM_ISR(uxSavedInterruptStatus);
}
```

### 2.3 临界区使用原则

| 规则 | 说明 |
|------|------|
| 尽可能短 | 临界区时间越短越好 |
| 不能阻塞 | 临界区内不能调用任何可能阻塞的API |
| 不能调用vTaskDelay | 包括其ISR版本 |
| 避免长操作 | 只保护必要的共享变量访问 |

## 三、中断优先级配置

### 3.1 ARM Cortex-M 配置

```c
// FreeRTOSConfig.h 中的配置
#define configPRIO_BITS            __NVIC_PRIO_BITS    // MCU的中断优先级位数
#define configKERNEL_INTERRUPT_PRIORITY    (255)       // 最低优先级（内核级）
#define configMAX_SYSCALL_INTERRUPT_PRIORITY (191)    // 系统调用最高优先级
```

**优先级分组**：
```
高优先级数值（0~configMAX_SYSCALL_INTERRUPT_PRIORITY-1）
    ↑
    |  不受FreeRTOS管理，完全禁用
    |  可在FreeRTOS临界区中执行
    |
    +--- configMAX_SYSCALL_INTERRUPT_PRIORITY
    |
    |  FreeRTOS管理的中断
    |  可在ISR中使用FromISR API
    |
低优先级数值
```

### 3.2 原则

- `configMAX_SYSCALL_INTERRUPT_PRIORITY` 以下的数值（优先级更高）的中断不受FreeRTOS影响
- FreeRTOS API只能在中断优先级 >= `configMAX_SYSCALL_INTERRUPT_PRIORITY` 的ISR中调用
- 关键时序要求高的中断应设置为高优先级（数值小）

## 四、Tickless Idle 模式

### 4.1 概述

Tickless模式是FreeRTOS的低功耗机制。在传统模式下，tick中断（通常1ms一次）周期性唤醒CPU，即使系统无事可做也会消耗功耗。

**Tickless模式**：
1. 空闲任务检测到所有任务都处于阻塞状态
2. 计算下一个需要唤醒的时间点（最早到期的定时器/任务延时）
3. 配置低功耗定时器在该时间点唤醒
4. 关闭tick中断，CPU进入低功耗模式
5. 低功定时器到期唤醒系统，恢复tick

### 4.2 启用

```c
// FreeRTOSConfig.h
#define configUSE_TICKLESS_IDLE    1
```

### 4.3 平台支持

不同MCU需要实现特定的低功耗函数：

```c
// 移植层需要实现的函数
void vPortSuppressTicksAndSleep(TickType_t xExpectedIdleTime);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `xExpectedIdleTime` | `TickType_t` | 预计睡眠时间（tick数），由空闲任务计算得出 |

FreeRTOS官方移植（如ARM Cortex-M）已经提供了此函数。

### 4.4 工作流程

```
时间轴:
|--任务A--| 阻塞 |    空闲/低功耗    |--任务B到期--| 阻塞 |  空闲/低功耗
          ^                        ^                         ^
          |进入低功耗             |唤醒                      |进入低功耗
          关闭tick中断            恢复tick                   关闭tick中断
```

### 4.5 注意事项

| 注意事项 | 说明 |
|----------|------|
| tick中断停止 | 低功耗期间 `xTaskGetTickCount()` 不会递增 |
| 唤醒源 | 需确保所有唤醒源都能正确触发 |
| 外设状态 | 进入低功耗前需处理外设状态 |
| 最大睡眠时间 | 取决于低功耗定时器的最大范围 |

## 五、完整示例

### 示例：综合低功耗中断管理

```c
static QueueHandle_t xDataQueue;
static TaskHandle_t xProcessTaskHandle;
static TimerHandle_t xPeriodicTimer;

// 传感器采集中断
void Sensor_IRQHandler(void)
{
    BaseType_t xHigherPriorityTaskWoken = pdFALSE;
    uint32_t data = read_sensor_data();

    // 中断安全方式发送到队列
    xQueueSendFromISR(xDataQueue, &data, &xHigherPriorityTaskWoken);

    portYIELD_FROM_ISR(xHigherPriorityTaskWoken);
}

// 周期性处理定时器回调
void vPeriodicCallback(TimerHandle_t xTimer)
{
    // 设置事件位通知处理任务
    xEventGroupSetBitsFromISR(xEventGroup, PERIODIC_EVENT_BIT, NULL);
}

// 数据处理任务
void vProcessTask(void *pvParameters)
{
    uint32_t data;
    EventBits_t bits;

    while (1) {
        // 等待队列数据或周期性事件
        BaseType_t has_data = xQueueReceive(xDataQueue, &data, 0);
        if (has_data) {
            process_data(data);
        }

        bits = xEventGroupWaitBits(
            xEventGroup, PERIODIC_EVENT_BIT,
            pdTRUE, pdFALSE, pdMS_TO_TICKS(100)
        );
        if (bits & PERIODIC_EVENT_BIT) {
            periodic_maintenance();
        }
    }
}

// 空闲钩子 - 进入低功耗
void vApplicationIdleHook(void)
{
    // 如果启用了 configUSE_TICKLESS_IDLE，
    // 这里会自动进入低功耗
    __WFI();  // Wait For Interrupt
}
```

| 函数 | 说明 |
|------|------|

| `taskENTER_CRITICAL_FROM_ISR()` | 中断中进入临界区 |
| `taskEXIT_CRITICAL_FROM_ISR()` | 中断中退出临界区 |

| 函数 | 说明 |

# 网络端口基础

## 一、什么是端口

端口是操作系统用来区分不同网络服务的**逻辑编号**，范围 0-65535。它与 IP 地址组合成完整的网络地址：`IP:端口`，例如 `192.168.1.100:1883`。

### 1.1 类比理解

```
IP 地址 = 一栋大楼的地址（比如 "中山路 100 号"）
端口号 = 大楼里的房间号（比如 "301 室"）

快递送到中山路 100 号后，还需要知道送到哪个房间。
网络数据包到达设备后，操作系统根据端口号分发给对应的程序。
```

### 1.2 端口分类

| 范围 | 名称 | 说明 |
|------|------|------|
| 0-1023 | **知名端口** | 系统级服务保留，需要 root/管理员权限绑定 |
| 1024-49151 | **注册端口** | 应用程序注册使用，无需特权 |
| 49152-65535 | **动态/私有端口** | 操作系统动态分配给客户端临时使用 |

### 1.3 TCP vs UDP 端口

- **TCP 端口**：面向连接，可靠传输（如 HTTP 80、MQTT 1883）
- **UDP 端口**：无连接，不可靠但快速（如 DNS 53、NTP 123）
- TCP 和 UDP 的端口号是**独立的**，TCP 80 和 UDP 80 是两个完全不同的端口

```
TCP 80 ──→ HTTP 服务
UDP 80 ──→ 完全不同的服务（如果有的话）
两者互不干扰
```

### 2.5 ESP32 常见端口使用

```
ESP32 作为 MQTT 客户端 → 连接到 Broker 的 1883 端口
ESP32 作为 Web Server  → 监听 80 端口
ESP32 作为 OTA 接收端  → 通常 3232 或自定义端口
ESP32 mDNS 响应        → UDP 5353
ESP32 NTP 时间同步     → UDP 123
ESP32 SNTP             → UDP 123（ESP-IDF 内置）
```

## 三、端口相关操作

### 3.1 查看端口占用

```bash
# Windows
netstat -ano | findstr :1883
# 输出: TCP  0.0.0.0:1883  0.0.0.0:0  LISTENING  12345
#                                              └─ PID

# 用 PID 到任务管理器找进程
tasklist | findstr 12345

# Linux
lsof -i :1883
ss -tlnp | grep 1883
```

### 3.2 测试端口连通性

```bash
# telnet（Windows/Linux 通用）
telnet 192.168.1.100 1883

# nc (netcat)
nc -zv 192.168.1.100 1883

# PowerShell
Test-NetConnection -ComputerName 192.168.1.100 -Port 1883
```

### 3.3 端口被占用处理

```bash
# Windows 查看占用
netstat -ano | findstr :1883
# 用 PID 到任务管理器找进程

# 解决
taskkill /PID <PID> /F
```

### 3.4 防火墙规则

```
入站规则（Inbound）：允许外部连接到本机的端口
出站规则（Outbound）：允许本机连接到外部的端口

ESP32 作为客户端 → 只需要出站规则放行
ESP32 作为服务器 → 需要入站规则放行监听端口
```

# 网络调试工具

## 一、ping —— 测试网络连通性

**类比**：
```
ping 就像你朝山谷喊一声"喂——"，然后听有没有回声。
- 有回声 = 对方在，网络通
- 没回声 = 对方不在或路不通
- 回声来得快 = 距离近，延迟低
- 回声来得慢 = 距离远，延迟高
```

```bash
# Windows
ping 192.168.1.1
ping 8.8.8.8 -t          # 持续 ping（Ctrl+C 停止）

# Linux
ping 192.168.1.1 -c 4    # ping 4 次
ping 192.168.1.1 -i 0.2  # 间隔 0.2 秒

# 输出解读：
# 64 bytes from 192.168.1.1: icmp_seq=1 ttl=64 time=3.2 ms
#                                          └── RTT 延迟
```

**延迟参考**：
- `< 1ms`：局域网（同一 WiFi）
- `1-20ms`：同城服务器
- `20-100ms`：国内跨省
- `100-300ms`：跨国

## 二、netstat —— 查看网络连接

**类比**：
```
netstat 就像查看邮局的工作记录：
- 谁在和谁通信？
- 用的是哪个窗口（端口）？
- 信是在处理中还是已经处理完了？
```

```bash
# Windows
netstat -an                # 显示所有连接
netstat -anb               # 显示对应进程名（需要管理员）

# 输出：
# TCP    0.0.0.0:1883    0.0.0.0:0    LISTENING  (Mosquitto)
# TCP    192.168.1.100:50001  13.212.94.125:1883  ESTABLISHED
#   └─ 本地IP:本地端口            远端IP:远端端口    状态
```

### 常见连接状态

| 状态 | 含义 | 类比 |
|------|------|------|
| `LISTENING` | 监听中，等待连接 | 邮局窗口开着，等客户来 |
| `ESTABLISHED` | 连接已建立，通信中 | 两个人正在通话 |
| `TIME_WAIT` | 连接刚关闭，等待清理 | 通话结束，还没挂电话 |
| `CLOSE_WAIT` | 对方已关闭，我这边还没关 | 对方挂了，我还没挂 |

```bash
# Linux 等价命令
netstat -tnp
ss -tnp
```

## 三、ipconfig / ifconfig —— 查看本机网络信息

```bash
# Windows
ipconfig /all

# 输出关键字段：
# 无线局域网适配器 WLAN:
#    IPv4 地址. . . . . . : 192.168.1.100
#    子网掩码 . . . . . . : 255.255.255.0
#    默认网关. . . . . .  : 192.168.1.1
#    DHCP 服务器 . . . .  : 192.168.1.1
#    DNS 服务器 . . . .  : 192.168.1.1

# Linux
ip addr show
ip route show           # 查看路由表
ip neigh show           # 查看 ARP 表
```

## 四、nslookup / dig —— 查询 DNS

```bash
# Windows
nslookup broker.emqx.io

# 输出：
# 服务器:  router
# Address:  192.168.1.1
#
# 非权威应答:
# 名称:    broker.emqx.io
# Address:  13.212.94.125

# Linux (更强大)
dig broker.emqx.io +short     # 简洁输出
dig broker.emqx.io ANY        # 查询所有记录
```

## 七、tracert / traceroute —— 追踪路由

**类比**：
```
你想知道从你家到某个地方经过了哪些路口。
tracert 会告诉你每一跳的 IP 地址和延迟。
```

```bash
# Windows
tracert 8.8.8.8

# Linux
traceroute 8.8.8.8

# 输出：每一行代表经过的一个路由器
```

## 八、抓包分析（Wireshark / tcpdump）

**类比**：
```
抓包就像把网络上的所有数据包都"录"下来，然后一帧一帧地看。
你想知道网络到底出了什么问题，抓包是最直接的方式。
```

```bash
# tcpdump 命令行抓包
tcpdump -i any port 1883          # 抓取 1883 端口的所有流量
tcpdump -i any host 192.168.1.100 # 抓取某个 IP 的所有流量

# Wireshark 图形界面抓包
# 1. 打开 Wireshark
# 2. 选择网卡（WiFi 或以太网）
# 3. 开始捕获
# 4. 过滤: tcp.port == 1883 或 mqtt
# 5. 查看 MQTT CONNECT、PUBLISH 等报文
```

### Wireshark 分析 MQTT

```
1. 选择 WiFi 网卡
2. 过滤: tcp.port == 1883
3. 右键 → Decode As → MQTT
4. 看到 CONNECT、PUBLISH、SUBSCRIBE 等报文
```

```
// 正常连接日志：

W mqtt_client: MQTT_EVENT_DISCONNECTED
E mqtt_client: MQTT publish failed

// 连接失败：
E mqtt_client: MQTT_EVENT_ERROR
E mqtt_client: esp_mqtt_client_publish message len=0
```

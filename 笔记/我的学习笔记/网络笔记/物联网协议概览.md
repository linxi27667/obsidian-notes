# 物联网协议概览

## 一、物联网网络架构

**类比**：
```
物联网像一个快递系统：

┌──────────────────────────────────────────────┐
│              云平台 / 服务器（快递公司总部）   │
│    ┌──────────┐  ┌──────────┐  ┌──────────┐  │
│    │ MQTT     │  │ Web      │  │ 数据库   │  │
│    │ Broker   │  │ 服务器   │  │ (MySQL)  │  │
│    └──────────┘  └──────────┘  └──────────┘  │
└──────────────┬───────────────┬───────────────┘
               │ TCP 1883     │ HTTP/WS
               │              │
┌──────────────┼──────────────┼───────────────┐
│    网络层    │  快递运输路线（WiFi/4G/5G）    │
└──────────────┼──────────────┼───────────────┘
               │              │
┌──────────────┼──────────────┼───────────────┐
│    感知层    │              │               │
│  ┌──────────┴──┐  ┌────────┴──────┐        │
│  │  ESP32 主控 │  │  ESP32 从设备 │        │
│  │  (驿站站长)  │  │  (快递员)     │        │
│  └─────────────┘  └───────────────┘        │
└────────────────────────────────────────────┘
```

### 典型 IoT 三层架构

```
感知层（设备层）：传感器、执行器、ESP32
网络层（传输层）：WiFi、以太网、4G/5G、LoRa
应用层（平台层）：云平台、数据库、Web 界面
```

---

## 二、IoT 通信模式

| 模式 | 类比 | 典型协议 |
|------|------|----------|
| **发布/订阅** | 你在公告板贴便条，感兴趣的人来看 | MQTT |
| **请求/响应** | 你打电话问"天气怎么样"，对方回答 | HTTP, CoAP |
| **推送通知** | 对方主动给你发消息，不用你问 | WebSocket |
| **设备到设备** | 两个快递员直接对讲，不需要通过驿站 | ESP-NOW, BLE |

## 三、IoT 专用协议

### 3.1 ESP-NOW

**类比**：
```
两个对讲机直接对话，不需要基站。
距离近（~100米），但速度极快，不用联网也能用。
```

```
特点: WiFi 直连、无需路由器、极低延迟、ESP32 专属
范围: ~100m（开阔地）
```

**特性**：
- 无需连接 WiFi AP，设备直连
- 单向/双向通信
- 支持加密（可选）
- 最大载荷 250 字节
- 一对多广播

**适用**：
- ESP32 设备间近距离通信
- 按键/传感器数据快速传输
- 不需要互联网的场景

---

### 3.2 BLE GATT（低功耗蓝牙）

**类比**：
```
你和手机之间有一条私密的小通道，距离很近（~10米），
非常省电，就像两个人悄悄说话。
```

```
范围: ~10m
特点: 极低功耗、短距离、手机直连
```

**架构**：
```
GATT Server (ESP32 外设)
  ├── Service A (如电池服务)
  │   ├── Characteristic: 电量百分比 (Read/Notify)
  │   └── Characteristic: 报警阈值 (Read/Write)
  └── Service B (如传感器服务)
      └── Characteristic: 温度值 (Notify)

GATT Client (手机 App)
  → 连接 → 发现服务 → 读取/写入/订阅特征值
```

---

### 3.3 LoRaWAN

**类比**：
```
一个超大功率的喇叭，声音能传到几公里外，
但只能传很少的内容（几个字），而且很慢。
适合远距离、数据量小的场景。
```

```
范围: ~1-10km（远距离）
速率: 极低（几 Kbps）
特点: 超低功耗、广域、星型拓扑
```

**适用**：
- 农业传感器（田间几公里范围）
- 城市基础设施监控
- 不需要实时性、数据量极小的场景

## 四、完整协议栈（小智 IoT 项目视角）

```
┌──────────────────────────────────────────────────┐
│  应用层                                           │
│  MQTT │ HTTP │ WebSocket │ CoAP │ DNS            │
├──────────────────────────────────────────────────┤
│  传输层                                           │
│  TCP (MQTT, HTTP, WS) │ UDP (DNS, CoAP, NTP)    │
├──────────────────────────────────────────────────┤
│  网络层                                           │
│  IPv4 │ ICMP (ping) │ ARP                       │
├──────────────────────────────────────────────────┤
│  数据链路层                                       │
│  WiFi (802.11) │ 以太网 (802.3) │ ESP-NOW       │
├──────────────────────────────────────────────────┤
│  物理层                                           │
│  2.4GHz 无线电 │ 网线 │ 光纤                    │
└──────────────────────────────────────────────────┘
```

## 五、常见场景的协议组合

| 场景 | 协议栈 |
|------|--------|
| ESP32 MQTT 通信 | WiFi → TCP → MQTT |
| ESP32 HTTPS 请求 | WiFi → TCP → TLS → HTTP |
| ESP32 WebSocket AI 对话 | WiFi → TCP → TLS → WebSocket |
| DNS 解析 | WiFi → UDP → DNS |
| NTP 时间同步 | WiFi → UDP → NTP |
| ESP32 直连另一个 ESP32 | WiFi (ESP-NOW) |
| 手机蓝牙连接 ESP32 | BLE → GATT |
| ping 测试 | WiFi → IP → ICMP |
| 浏览器查看 ESP32 Web | WiFi → TCP → HTTP |

## 六、协议选择决策树

```
需要传输数据？
  │
  ├─ 设备间近距离、无需网络？
  │   └─→ ESP-NOW / BLE
  │
  ├─ 需要发布/订阅、一对多？
  │   └─→ MQTT（推荐）/ CoAP（超低功耗）
  │
  ├─ 浏览器交互？
  │   ├─ 请求/响应 → HTTP/HTTPS
  │   └─ 实时双向 → WebSocket
  │
  ├─ 文件传输？
  │   └─→ FTP/SFTP/HTTP
  │
  ├─ 时间同步？
  │   └─→ NTP
  │
  └─ 域名解析？
      └─→ DNS
```

## 七、IoT 协议对比总结

| 协议 | 传输层 | 开销 | QoS | 适用场景 |
|------|--------|------|-----|----------|
| **MQTT** | TCP | 最小 2 字节 | 3 级 | IoT 设备通信（推荐） |
| **HTTP** | TCP | 大（Headers 多） | 无 | Web API、REST |
| **CoAP** | UDP | 小 | 4 级 | 低功耗受限设备 |
| **WebSocket** | TCP | 中等 | 无 | 浏览器实时通信 |
| **ESP-NOW** | WiFi | 极小 | 无 | ESP32 设备直连 |
| **BLE GATT** | BLE | 小 | 无 | 短距离低功耗 |
| **LoRaWAN** | LoRa | 极小 | 无 | 远距离小数据 |

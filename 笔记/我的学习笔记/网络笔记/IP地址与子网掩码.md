# IP 地址与子网掩码

## 一、IPv4 地址结构

IPv4 地址由 **4 个字节（32 位）** 组成，通常写成十进制点分格式：

```
192 . 168 .  1  . 100
 ▲     ▲    ▲    ▲
 │     │    │    │
 │     │    │    └─ 第 4 段（8位）= 01100100
 │     │    └────── 第 3 段（8位）= 00000001
 │     └─────────── 第 2 段（8位）= 10101000
 └───────────────── 第 1 段（8位）= 11000000
```

每一段范围：**0 ~ 255**（8 位二进制，2⁸ = 256 个值）。

---

## 二、网络号与主机号

IP 地址本质上分为两部分：**网络号 + 主机号**

```
IP 地址 = 网络号（标识在哪个网段） + 主机号（标识该网段内的具体设备）
```

以 `192.168.1.100` 为例：
- 前 3 段 `192.168.1` 是**网络号**（标识这是 192.168.1.x 这个局域网）
- 最后 1 段 `.100` 是**主机号**（标识这个网段内第 100 号设备）

**类比**：IP 地址像电话号码 `区号-号码`
- `192.168.1` = 区号（确定在哪个城市/网段）
- `.100` = 具体号码（确定哪个人/设备）

### 二进制视角

```
192.168.1.100
= 11000000.10101000.00000001.01100100
  ────8位─── ────8位─── ────8位─── ────8位───
```

网络层做路由判断时，实际是对二进制位做**按位与运算**。

## 三、特殊 IP 地址

| IP 地址 | 用途 | 示例 |
|---------|------|------|
| `127.0.0.1` | 本地回环（localhost） | 测试本机网络协议栈 |
| `0.0.0.0` | 所有地址 / 未指定 | 服务器监听所有网卡 |
| `255.255.255.255` | 受限广播地址 | 向本网段所有设备发送 |
| `192.168.1.255` | 定向广播地址 | 向 192.168.1.x 网段广播 |
| `169.254.x.x` | APIPA（自动私有IP） | DHCP 失败时系统自动分配 |
| `224.0.0.0 ~ 239.255.255.255` | 组播地址 | 一对多通信 |

## 四、子网掩码（Subnet Mask）

### 4.1 子网掩码是什么

子网掩码用来告诉设备：**IP 地址中哪些位是网络号，哪些位是主机号**。

规则：
- 掩码中 **1** 对应的位 → 网络号
- 掩码中 **0** 对应的位 → 主机号

### 4.2 常见子网掩码

| 掩码（十进制） | 掩码（二进制） | CIDR | 可用主机数 | 典型场景 |
|---------------|---------------|------|-----------|---------|
| `255.0.0.0` | `11111111.00000000.00000000.00000000` | /8 | 16,777,214 | A 类大网络 |
| `255.255.0.0` | `11111111.11111111.00000000.00000000` | /16 | 65,534 | B 类中型网络 |
| `255.255.255.0` | `11111111.11111111.11111111.00000000` | /24 | 254 | **家用路由器（最常见）** |
| `255.255.255.128` | `11111111.11111111.11111111.10000000` | /25 | 126 | 子网划分 |
| `255.255.255.192` | `11111111.11111111.11111111.11000000` | /26 | 62 | 子网划分 |
| `255.255.255.252` | `11111111.11111111.11111111.11111100` | /30 | 2 | 点对点链路 |

### 4.3 计算示例

**已知**：
- IP：`192.168.1.100`
- 掩码：`255.255.255.0`（/24）

**计算**：

```
IP:       192.168.1.100  =  11000000.10101000.00000001.01100100
掩码:     255.255.255.0  =  11111111.11111111.11111111.00000000
          ─────────────     ───────────────────────────────
网络地址: 192.168.1.0    =  11000000.10101000.00000001.00000000
           (IP AND 掩码)
```

**推导**：

| 项目 | 值 | 说明 |
|------|-----|------|
| 网络地址 | `192.168.1.0` | 该网段的起始地址 |
| 广播地址 | `192.168.1.255` | 主机位全为 1 |
| 可用 IP 范围 | `192.168.1.1` ~ `192.168.1.254` | 排除网络地址和广播地址 |
| 可用主机数 | 254 | 2⁸ - 2 = 256 - 2 |

> **为什么要减 2？** 网络地址（全 0）和广播地址（全 1）不能分配给设备。

### 4.4 判断两台设备是否在同一网段

```
设备 A: 192.168.1.100 /24
设备 B: 192.168.1.200 /24
设备 C: 192.168.2.100 /24

判断方法：各自 IP AND 掩码，结果相同 = 同一网段

A AND 255.255.255.0 = 192.168.1.0
B AND 255.255.255.0 = 192.168.1.0  ← 与 A 相同 ✅ 可以直接通信
C AND 255.255.255.0 = 192.168.2.0  ← 与 A 不同 ❌ 需要路由器
```

## 五、CIDR（无类域间路由）

CIDR 用斜杠后面跟数字表示掩码长度：

```
192.168.1.0/24    ← 前 24 位是网络号
10.0.0.0/8        ← 前 8 位是网络号
172.16.0.0/12     ← 前 12 位是网络号
```

**读法**：`/24` 表示掩码有 24 个连续的 1，即 `255.255.255.0`。

### 快速计算公式

```
主机数 = 2^(32 - 前缀长度) - 2

/24 → 2^(32-24) - 2 = 2^8 - 2 = 254
/26 → 2^(32-26) - 2 = 2^6 - 2 = 62
/28 → 2^(32-28) - 2 = 2^4 - 2 = 14
/30 → 2^(32-30) - 2 = 2^2 - 2 = 2（点对点链路刚好）
```

## 六、子网划分练习

### 将 /24 划分为 4 个子网

```
原始: 192.168.1.0/24 (256 个地址)

借用 2 位主机位作为子网位 → /26

子网 0: 192.168.1.0/26     范围 .1 ~ .62     (62台)
子网 1: 192.168.1.64/26    范围 .65 ~ .126   (62台)
子网 2: 192.168.1.128/26   范围 .129 ~ .190  (62台)
子网 3: 192.168.1.192/26   范围 .193 ~ .254  (62台)

二进制观察：
.0     = 00 000000  ← 子网位 00
.64    = 01 000000  ← 子网位 01
.128   = 10 000000  ← 子网位 10
.192   = 11 000000  ← 子网位 11
```

## 七、网关（Gateway）

### 7.1 网关是什么

网关是**连接不同网络的出口设备**（通常是路由器）。

```
你的电脑: 192.168.1.100/24
默认网关: 192.168.1.1（路由器的 LAN 口）

访问 192.168.1.50   → 同一网段 → 直接发送（ARP 请求）
访问 8.8.8.8        → 不同网段 → 发给网关 192.168.1.1
                              → 路由器做 NAT 转发到互联网
```

### 7.2 默认网关的工作原理

```
设备发送数据包时的判断逻辑：

1. 目标 IP AND 本机掩码 = ?
2. 结果 == 本机网络号？
   ├─ 是 → 本地交付（ARP 查找目标 MAC，直接发送）
   └─ 否 → 发给默认网关（路由器处理）
```

### 7.3 ESP32 中的网关配置

```c
// ESP-IDF 静态 IP 配置示例
esp_netif_t* netif = esp_netif_get_handle_from_ifkey("WIFI_STA_DEF");
esp_netif_ip_info_t ip_info;
ip_info.ip.addr      = esp_ip4addr_aton("192.168.1.100");   // 本机 IP
ip_info.netmask.addr = esp_ip4addr_aton("255.255.255.0");    // 子网掩码
ip_info.gw.addr      = esp_ip4addr_aton("192.168.1.1");      // 默认网关
esp_netif_set_ip_info(netif, &ip_info);
```

## 八、路由表

### 8.1 路由表是什么

路由表决定**数据包从哪个接口/网关发出**。

```
# Linux 路由表示例
Destination     Gateway         Genmask         Iface
0.0.0.0         192.168.1.1     0.0.0.0         wlan0    ← 默认路由
192.168.1.0     0.0.0.0         255.255.255.0   wlan0    ← 本地网段
127.0.0.0       0.0.0.0         255.0.0.0       lo       ← 回环
```

- `0.0.0.0/0` 是默认路由（匹配所有目标），指向网关
- `192.168.1.0/24` 是直连路由（同一网段直接发送）

### 8.2 路由匹配原则

```
目标 IP: 192.168.1.50
  → 匹配 192.168.1.0/24（最长前缀匹配）→ 直接从 wlan0 发送

目标 IP: 8.8.8.8
  → 不匹配任何具体路由 → 走默认路由 0.0.0.0/0 → 发给 192.168.1.1
```

## 十、DHCP vs 静态 IP

| | DHCP（自动） | 静态 IP（手动） |
|------|------|------|
| 配置 | 路由器自动分配 | 手动设置 |
| 适用 | 客户端设备 | 服务器、网关 |
| 优点 | 即插即用 | 地址固定 |
| 缺点 | 地址可能变化 | 需要手动管理 |
| ESP32 使用 | 默认模式 | `esp_netif` 手动配置 |

### ESP32 WiFi 连接代码模板

```c
// ESP-IDF WiFi 连接（STA 模式）
wifi_config_t wifi_config = {
    .sta = {
        .ssid = "YourWiFiSSID",
        .password = "YourPassword",
    },
};
esp_wifi_connect();
// 等待 IP_EVENT_STA_GOT_IP 事件
```

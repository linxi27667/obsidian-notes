# RS-485 （差分信号 多设备总线 半双工）

## 核心概念

### 电平标准
- ==差分传输== - A线与B线电压差表示逻辑
- ==逻辑1== - A-B > +200mV（MARK）
- ==逻辑0== - B-A > +200mV（SPACE）
- ==抗干扰== - 差分信号抵消共模噪声

### 通信特性
- ==最大距离== - 1200米（低速率）
- ==最大速率== - 10Mbps（短距离）
- ==多设备== - 最多32个标准节点，最多256个（带扩展）
- ==半双工== - 通常两线制，需方向控制

## 一、硬件配置

### 1.1 两线制接线

```
       STM32 Master
          │
     ┌────┴────┐
     │  MAX485 │
     │   A  B  │
     └──┬──┬───┘
        │  │
    ┌───┴──┴───┐   ┌───┴──┴───┐   ┌───┴──┴───┐
    │ MAX485   │   │ MAX485   │   │ MAX485   │
    │ Slave1   │   │ Slave2   │   │ Slave3   │
    └──────────┘   └──────────┘   └──────────┘
    
    A线 ──────────── 所有设备A端连接
    B线 ──────────── 所有设备B端连接
    
注意：总线两端需要120Ω终端电阻
```

### 1.2 MAX485芯片接口

| 引脚 | 名称 | 功能 |
|------|------|------|
| 1 | RO | 接收输出（接STM32 RX） |
| 2 | RE | 接收使能（低有效） |
| 3 | DE | 发送使能（高有效） |
| 4 | DI | 发送输入（接STM32 TX） |
| 5 | GND | 地 |
| 6 | A | 差分线A |
| 7 | B | 差分线B |
| 8 | VCC | +5V供电 |

### 1.3 STM32接线示意

| STM32引脚 | MAX485引脚 | 功能 |
|-----------|-----------|------|
| UART_TX (PA9) | DI (4) | 发送输入 |
| UART_RX (PA10) | RO (1) | 接收输出 |
| GPIO (PA8) | DE+RE (3+2) | 方向控制 |

**方向控制逻辑**：
- GPIO = HIGH：发送模式（DE=1, RE=1）
- GPIO = LOW：接收模式（DE=0, RE=0）

## 二、RS485驱动流程

### 2.1 初始化步骤

1. **UART初始化**：波特率9600（RS485常用），8N1
2. **方向控制GPIO初始化**：推挽输出模式
3. **默认状态**：设为接收模式（DE=0, RE=0）

### 2.2 发送流程

1. 切换为发送模式（DE=1, RE=1）
2. 等待短暂延迟确保电平稳定
3. 通过 UART 发送数据
4. 等待 UART TC（发送完成）标志位置位
5. 切换回接收模式

### 2.3 接收方式

- **阻塞接收**：`HAL_UART_Receive` 等待指定长度数据
- **中断接收**：`HAL_UART_Receive_IT` 逐字节中断回调
- 接收前确保处于接收模式（DE=0, RE=0）

## 三、多设备通信协议

### 3.1 主从架构

> [!NOTE] 通信规则
> - 主站发起通信，从站响应
> - 从站有唯一地址（1~247）
> - 从站不能主动发送

**数据帧格式**：

| 字段 | 长度 | 说明 |
|------|------|------|
| 从站地址 | 1 byte | 目标从站（0为广播） |
| 功能码 | 1 byte | 操作类型 |
| 数据 | N bytes | 载荷数据 |
| CRC | 2 bytes | CRC16校验（低字节先发） |

### 3.2 从站响应流程

1. 接收数据并检查地址匹配
2. 解析功能码并分发到对应处理函数
3. 构造响应帧（含CRC）
4. 切换发送模式并响应
5. 切回接收模式等待下次请求

## 四、硬件设计要点

### 4.1 终端电阻

> [!TIP] 终端电阻
> 总线两端各加一个120Ω电阻，消除信号反射

**接法说明**：
- 仅在总线**最远两端**加120Ω电阻（A线与B线之间）
- 中间节点**不加**终端电阻
- 终端电阻匹配总线特性阻抗，防止高速信号反射造成干扰

### 4.2 常用RS485芯片对比

| 芯片 | 速率 | 供电 | 特点 |
|------|------|------|------|
| MAX485 | 2.5Mbps | 5V | 经典，便宜 |
| MAX3485 | 3Mbps | 3.3V | STM32兼容 |
| SP3485 | 10Mbps | 3.3V | 高速 |
| SN65HVD75 | 1Mbps | 3.3V | 低功耗 |

## 五、主站轮询策略

### 5.1 轮询流程

1. 依次向各从站发送读取请求
2. 等待从站响应（设置超时）
3. 校验响应CRC
4. 解析数据并处理
5. 延迟后轮询下一个从站（帧间隔 > 3.5字符时间）

## 六、常见问题排查

### 6.1 通信不稳定

**检查清单**：
1. 终端电阻是否正确安装在总线两端
2. A/B线是否接反
3. 波特率是否一致
4. 方向控制时序是否正确（发送前切TX，发送完成后切RX）
5. 是否有电气干扰

**方向切换时序优化**：
- 先设置GPIO为发送模式
- 等待UART TC标志（传输完成）
- 再开始发送数据
- 发送完成后等待最后一个字节送出再切回接收

### 6.2 数据帧冲突

> [!WARNING] 冲突原因
> 多个从站同时响应会导致总线冲突

**解决方案**:
- 严格遵守主从协议
- 从站只在收到正确地址的请求后响应
- 设置合理的响应超时时间

# RS-232 （负逻辑电平 长距离串口 DCE/DTE）

## 核心概念

### 电平标准
- ==负逻辑== - 逻辑1为负电压，逻辑0为正电压
- ==电压范围== - 数据线 ±3V~±15V，控制线 ±3V~±25V
- ==逻辑1== - -3V ~ -15V（MARK）
- ==逻辑0== - +3V ~ +15V（SPACE）

### 通信特性
- ==最大距离== - 约15米（标准）
- ==最大速率== - 20kbps（标准），实际可达115200
- ==点对点== - 一对一通信

## 一、电平转换芯片

### 1.1 MAX232芯片

> [!TIP] MAX232系列
> 最常用的RS-232电平转换芯片，单5V供电

**接线示意**：
- STM32 TX → MAX232 T1IN → RS-232 TX（DB9 PIN 2）
- STM32 RX ← MAX232 R1OUT ← RS-232 RX（DB9 PIN 3）

**关键引脚**：
- T1IN (11)：TTL输入（发送）
- T1OUT (14)：RS232输出
- R1IN (13)：RS232输入
- R1OUT (12)：TTL输出（接收）
- C1+, C1-, C2+, C2-：外接电容（1μF）

### 1.2 常用电平转换芯片对比

| 芯片 | 供电 | 特点 |
|------|------|------|
| MAX232 | +5V | 经典，需4个电容 |
| MAX233 | +5V | 内置电容，节省空间 |
| MAX3232 | +3V~5V | 3.3V兼容 |
| SP3232 | +3V~5V | 低功耗 |

## 二、DB9接口定义

### 2.1 公头/母头定义

```
DB9公头（DTE端，如电脑）:
┌─────────────────┐
│ 1 2 3 4 5       │  1: DCD  6: DSR
│  6 7 8 9        │  2: RX   7: RTS
└─────────────────┘  3: TX   8: CTS
                     4: DTR  9: RI
                     5: GND
```

### 2.2 最小三线连接

| 信号 | DB9引脚 | 说明 |
|------|---------|------|
| TX | 3 | 发送数据 |
| RX | 2 | 接收数据 |
| GND | 5 | 信号地 |

### 2.3 完整流控连接

| 信号 | DB9引脚 | 方向 | 说明 |
|------|---------|------|------|
| DCD | 1 | IN | 载波检测 |
| RX | 2 | IN | 接收数据 |
| TX | 3 | OUT | 发送数据 |
| DTR | 4 | OUT | 数据终端就绪 |
| GND | 5 | - | 信号地 |
| DSR | 6 | IN | 数据设备就绪 |
| RTS | 7 | OUT | 请求发送 |
| CTS | 8 | IN | 清除发送 |
| RI | 9 | IN | 振铃指示 |

## 三、硬件流控

### 3.1 RTS/CTS流控配置

- 初始化时设置 `HwFlowCtl = RTS_CTS`
- 需额外配置 RTS（PA12）和 CTS（PA11）引脚为复用功能
- 通过硬件信号线自动暂停/恢复数据流，避免缓冲区溢出

### 3.2 流控工作原理

- **RTS（Request To Send）**：发送方请求发送数据
- **CTS（Clear To Send）**：接收方确认可以接收数据
- 若CTS为低，发送方暂停发送，防止数据丢失
- 适用于高速率、大数据量传输场景

## 四、DCE与DTE接线

### 4.1 DTE-DCE连接（直通线）

> [!NOTE] 设备类型
> - DTE (Data Terminal Equipment)：数据终端设备，如电脑、STM32
> - DCE (Data Communication Equipment)：数据通信设备，如调制解调器

| DTE端 | DCE端 | 说明 |
|-------|-------|------|
| TX (3) | RX (2) | 发送对接收 |
| RX (2) | TX (3) | 接收对发送 |
| GND (5) | GND (5) | 共地 |

使用**直通线**（公头-母头）

### 4.2 DTE-DTE连接（交叉线）

| 设备A | 设备B | 说明 |
|-------|-------|------|
| TX (3) | RX (2) | 交叉发送 |
| RX (2) | TX (3) | 交叉接收 |
| GND (5) | GND (5) | 共地 |

使用**交叉线**（公头-公头），也称Null Modem线

## 四、DCE与DTE接线

### 4.1 DTE-DCE连接

> [!NOTE] 设备类型
> - DTE (Data Terminal Equipment): 数据终端设备，如电脑、STM32
> - DCE (Data Communication Equipment): 数据通信设备，如调制解调器

```
DTE（STM32）         DCE（调制解调器）
    TX (3) ──────────── RX (2)
    RX (2) ──────────── TX (3)
    GND (5)─────────── GND (5)
    
直通线连接（公头-母头）
```

### 4.2 DTE-DTE连接（交叉线）

```
STM32 A              STM32 B
    TX (3) ──────────── RX (2)
    RX (2) ──────────── TX (3)
    GND (5)─────────── GND (5)

需要交叉线（公头-公头）
```

## 五、完整STM32示例

### 5.1 基础通信示例

```c
// main.c
#include "main.h"
#include "usart.h"

UART_HandleTypeDef huart1;
uint8_t rx_buffer[64];

int main(void)
{
    HAL_Init();
    SystemClock_Config();
    MX_USART1_UART_Init();
    
    // 通过RS-232发送数据
    char *msg = "RS-232 Test from STM32\r\n";
    HAL_UART_Transmit(&huart1, (uint8_t*)msg, strlen(msg), HAL_MAX_DELAY);
    
    // 接收数据
    while(1)
    {
        if(HAL_UART_Receive(&huart1, rx_buffer, 1, 100) == HAL_OK)
        {
            // 回显
            HAL_UART_Transmit(&huart1, rx_buffer, 1, HAL_MAX_DELAY);
        }
    }
}
```

### 5.2 带流控的通信示例

```c
// 使用RTS/CTS硬件流控
void Send_With_FlowControl(uint8_t *data, uint16_t len)
{
    // 等待CTS信号（对方准备好接收）
    while(HAL_GPIO_ReadPin(GPIOA, GPIO_PIN_11) == GPIO_PIN_RESET);  // CTS低=忙
    
    // 发送数据
    HAL_UART_Transmit(&huart1, data, len, HAL_MAX_DELAY);
}

// RTS控制
void Set_RTS_Pin(GPIO_PinState state)
{
    // RTS高=可以接收，RTS低=不能接收
    HAL_GPIO_WritePin(GPIOA, GPIO_PIN_12, state);
}
```

## 六、常见问题与解决

### 6.1 电平不匹配

> [!WARNING] 问题
> STM32是TTL电平(0~3.3V)，RS-232是±12V，直接连接会损坏芯片

**解决方案**: 必须使用电平转换芯片（MAX232/MAX3232）

### 6.2 波特率误差

```c
// 检查实际波特率误差
// STM32 F1系列: PCLK2=72MHz时
// 115200bps: 实际115107, 误差0.08% (<2%可接受)

// 计算公式
// BRR = PCLK / (16 * BaudRate)
// 实际波特率 = PCLK / (16 * BRR)
```

### 6.3 接线错误排查

```
常见错误:
1. TX-TX、RX-RX直连（应交叉）
2. 公头母头搞混
3. GND未连接

排查步骤:
1. 测量MAX232输出端是否有±10V左右电压
2. 用示波器观察TX波形
3. 确认GND共地
```

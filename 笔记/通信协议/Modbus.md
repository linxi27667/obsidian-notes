# Modbus协议 （主从架构 功能码定义 CRC校验）

## 核心概念

### 协议概述
- ==主从架构== - 主站发起请求，从站响应
- ==功能码== - 定义操作类型（读/写）
- ==CRC校验== - RTU模式使用CRC16
- ==地址范围== - 1~247（0为广播）

### Modbus类型
| 类型 | 物理层 | 校验 | 特点 |
|------|--------|------|------|
| Modbus RTU | RS-485 | CRC16 | 工业，最常用 |
| Modbus ASCII | RS-485 | LRC | 可读性好，效率低 |
| Modbus TCP | Ethernet | 无 | 网络应用 |

## 一、Modbus RTU帧格式

### 1.1 请求帧结构

```
┌──────┬──────┬────────────┬──────┐
│地址  │功能码│数据        │CRC   │
│1byte│1byte │N bytes     │2byte │
└──────┴──────┴────────────┴──────┘

示例：读取从站1的寄存器地址0x0000，长度2
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│01   │03   │00   │00   │00   │02   │C4   │0B   │
│地址 │功能 │起始地址H│起始地址L│数量H│数量L│CRC_L│CRC_H│
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
```

### 1.2 响应帧结构

```
┌──────┬──────┬─────┬────────────┬──────┐
│地址  │功能码│字节 │数据        │CRC   │
│1byte│1byte │1byte│N bytes     │2byte │
└──────┴──────┴─────┴────────────┴──────┘

示例：读取响应
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│01   │03   │04   │00   │01   │00   │02   │XX   │XX   │
│地址 │功能 │字节数│数据1H│数据1L│数据2H│数据2L│CRC   │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
```

| 功能码 | 名称 | 操作对象 |
|--------|------|----------|
| 01 | 读线圈 | Coil（位） |

| 04 | 读输入寄存器 | Input Register（16位） |
| 05 | 写单个线圈 | Coil（位） |
| 06 | 写单个寄存器 | Holding Register（16位） |
| 15 | 写多个线圈 | Coil（位） |
| 16 | 写多个寄存器 | Holding Register（16位） |

### 2.2 功能码03请求帧结构

| 字段 | 长度 | 说明 |
|------|------|------|
| 从站地址 | 1 byte | 目标从站地址（1~247） |
| 功能码 | 1 byte | 0x03（读保持寄存器） |
| 起始地址高 | 1 byte | 起始寄存器地址高字节 |
| 起始地址低 | 1 byte | 起始寄存器地址低字节 |
| 寄存器数量高 | 1 byte | 要读取的寄存器数量高字节 |
| 寄存器数量低 | 1 byte | 要读取的寄存器数量低字节 |
| CRC低 | 1 byte | CRC16校验低字节 |
| CRC高 | 1 byte | CRC16校验高字节 |

**响应帧**：从站地址(1) + 功能码(1) + 字节数(1) + 数据(N×2) + CRC(2)

### 2.3 功能码06请求帧结构

| 字段 | 长度 | 说明 |
|------|------|------|
| 从站地址 | 1 byte | 目标从站地址 |
| 功能码 | 1 byte | 0x06（写单个寄存器） |
| 寄存器地址高 | 1 byte | 目标寄存器地址高字节 |
| 寄存器地址低 | 1 byte | 目标寄存器地址低字节 |
| 写入值高 | 1 byte | 写入数据高字节 |
| 写入值低 | 1 byte | 写入数据低字节 |
| CRC低 | 1 byte | CRC16校验低字节 |
| CRC高 | 1 byte | CRC16校验高字节 |

**响应帧**：与请求帧完全相同（从站回显确认写入成功）

## 三、CRC16校验

### 3.1 CRC16算法原理

- **多项式**：0xA001（反转后的 0x8005）
- **初始值**：0xFFFF
- **计算方式**：逐字节异或后逐位右移，若最低位为1则异或多项式
- **字节序**：低字节在前，高字节在后（Little-Endian）

### 3.2 校验流程

1. 初始化 CRC = 0xFFFF
2. 对每个数据字节：CRC ^= 字节值
3. 对每个位（8次）：若最低位为1，CRC = (CRC >> 1) ^ 0xA001；否则 CRC >>= 1
4. 最终 CRC 值附加到帧末尾，低字节先发

### 3.3 验证方法

接收方对整个帧（含CRC）重新计算 CRC16，若结果为 0 则校验通过。也可将收到的 CRC 分离后与计算值比较。

## 四、主站通信流程

### 4.1 主站读寄存器流程

1. **构造请求帧**：填充从站地址、功能码0x03、起始地址、寄存器数量、CRC
2. **切换RS485为发送模式**：拉高 DE/RE 引脚
3. **发送请求**：通过 UART 发送完整帧
4. **等待发送完成**：确认 UART TC 标志位置位
5. **切换RS485为接收模式**：拉低 DE/RE 引脚
6. **接收响应**：预期长度 = 3 + 寄存器数量×2 + 2（CRC）
7. **CRC校验**：验证响应完整性
8. **解析数据**：从响应帧中提取寄存器值（大端序：高字节在前）

### 4.2 主站写单个寄存器流程

1. **构造请求帧**：功能码0x06 + 寄存器地址 + 写入值 + CRC
2. **发送并切换接收**：同上
3. **接收响应**：响应帧与请求帧完全相同（回显确认）
4. **校验响应**：逐字节对比请求与响应，一致则写入成功

## 五、从站通信流程

### 5.1 从站寄存器区

从站需维护四类数据区：

| 数据区 | 读写属性 | 单位 | 对应功能码 |
|--------|----------|------|------------|
| 保持寄存器 | 可读可写 | 16位 | 03/06/16 |
| 输入寄存器 | 只读 | 16位 | 04 |
| 线圈 | 可读可写 | 1位 | 01/05/15 |
| 离散输入 | 只读 | 1位 | 02 |

### 5.2 从站请求处理流程

1. **接收请求帧**：等待至少8字节（最小帧长度）
2. **地址匹配**：检查从站地址是否匹配或为广播地址(0)
3. **CRC校验**：验证帧完整性，失败则丢弃
4. **功能码分发**：根据功能码调用对应处理函数
5. **范围检查**：验证请求的寄存器地址和数量是否越界
6. **构造响应**：填充数据并计算CRC
7. **发送响应**：RS485切换到发送模式，发送后切回接收

### 5.3 异常响应

当从站检测到错误时，返回异常帧：
- **功能码**：原功能码最高位置1（如 0x03 → 0x83）
- **错误码**：1字节，标识错误类型

| 错误码 | 名称 | 说明 |
|--------|------|------|
| 0x01 | ILLEGAL FUNCTION | 不支持的功能码 |
| 0x02 | ILLEGAL ADDRESS | 寄存器地址超出范围 |
| 0x03 | ILLEGAL VALUE | 数据值非法 |
| 0x04 | SLAVE FAILURE | 从站设备故障 |

## 六、主站轮询策略

### 6.1 多从站轮询

- 主站依次向各从站发送读取请求
- 每个从站响应后需等待帧间隔（>3.5字符时间）再发起下一次请求
- 超时处理：若从站未响应，标记通信错误并重试或跳过

### 6.2 写入控制命令

- 写单个寄存器（功能码06）用于设置参数阈值
- 写多个寄存器（功能码16）用于批量下发配置
- 写操作需验证回显响应，确认写入成功

## 七、帧间隔时序

### 7.1 Modbus RTU帧间隔要求

| 间隔类型 | 要求 | 说明 |
|----------|------|------|
| 帧内字节间隔 | < 1.5字符时间 | 同一帧内相邻字节最大间隔 |
| 帧间间隔 | > 3.5字符时间 | 两帧之间的最小间隔 |

### 7.2 字符时间计算

- 1字符 = 11位（1起始位 + 8数据位 + 1校验位 + 1停止位）
- 1字符时间 = 11 / 波特率（秒）
- **9600波特率**：3.5字符时间 ≈ 4ms
- **115200波特率**：3.5字符时间 ≈ 0.33ms

## 八、Modbus地址映射（触摸屏/HMI通信）

### 8.1 Modbus四种数据区（"旅馆房间"模型）

Modbus 将设备内部数据区比作四种类型的房间，每种房间有固定前缀编号：

| 前缀 | 数据区 | 读写属性 | 单位 | 典型应用 |
|------|--------|----------|------|----------|
| 0xxxx | 线圈（Coil） | 可读可写 | 1位 | 启动/停止按钮、继电器输出 |
| 1xxxx | 离散输入（Discrete Input） | 只读 | 1位 | 限位开关、传感器开关量 |
| 3xxxx | 输入寄存器（Input Register） | 只读 | 16位 | 传感器实时数据（ADC值） |
| 4xxxx | 保持寄存器（Holding Register） | 可读可写 | 16位 | 电机高度、目标速度、配置参数 |

### 8.2 "40001"地址解码

`40001` 是 Modicon 公司定义的**绝对地址**格式，拆解规则：

- **首位数字 `4`**：代表"保持寄存器"大区
- **后四位 `0001`**：代表该大区内的第 1 号房间

**⚠️ 底层地址偏移（关键避坑点）**：

| 触摸屏设置 | 协议底层传输 | 说明 |
|------------|-------------|------|
| 4x + 地址1 | 起始地址 0x0000 | 40001 → 底层地址减1 |
| 4x + 地址2 | 起始地址 0x0001 | 40002 → 底层地址减1 |
| 3x + 地址1 | 起始地址 0x0000 | 30001 → 同样减1 |

前缀数字（0/1/3/4）**不会**出现在电线上，它由功能码代替：
- `4x` → 功能码 `03`（读）或 `06/16`（写）
- `3x` → 功能码 `04`（读）
- `0x` → 功能码 `01`（读）或 `05/15`（写）
- `1x` → 功能码 `02`（读）

### 8.3 保持寄存器 vs 输入寄存器

| 对比项 | 保持寄存器 | 输入寄存器 |
|--------|-----------|-----------|
| 功能码 | 03（读）/ 06/16（写） | 04（只读） |
| 读写性 | 可读可写 | 只读 |
| 命名由来 | 数据写入后设备会"保持"该状态 | 实时输入数据，一直在变 |
| 典型用途 | 目标高度、速度设定值、配置参数 | 温度传感器读数、ADC采样值 |

### 8.4 "读取数量"的含义

- **数量 = 1**：读取 1 个寄存器（返回 2 字节）
- **数量 = 2**：读取 2 个连续寄存器（返回 4 字节）

**何时需要数量 > 1**：
- 16位数据（最大值65535）→ 数量1即可
- 32位浮点数（如 12.5034）→ 需占用2个寄存器，数量=2
- 32位整数（如编码器累计值1000000）→ 需占用2个寄存器，数量=2

### 8.5 触摸屏与MCU通信架构

```
触摸屏（HMI） ──── RS485总线 ──── MCU（STM32等）
   │                                │
 Modbus Master                   Modbus Slave
   │                                │
 主动轮询读取                     维护寄存器数组
 数值元件→4x地址                Reg[0]=电机高度
```

**通信角色**：
- **触摸屏 = Master（主站）**：主动发起读取请求，周期性轮询
- **MCU = Slave（从站）**：被动响应，将传感器数据写入寄存器数组

**物理层配置**（必须两端一致）：
- 波特率：9600（常用）
- 数据位：8
- 停止位：1
- 校验：None
- 物理芯片：MAX3485 / MAX485

### 8.6 HMI通信配置步骤

1. **通讯目标**：将数值元件的读取地址从 `HMIO`（屏幕本地）改为 `PLC_1`（外部设备）
2. **地址类型**：选择 `4x`（保持寄存器）
3. **地址编号**：填入寄存器号（如 `1` 对应底层地址 `0x0000`）
4. **小数位数**：在HMI端设置小数点（如2位），MCU发整数 `1250`，屏幕显示 `12.50`
5. **单位标签**：在HMI端添加右标签（如 `mm`），MCU无需处理单位

### 8.7 实际通信帧示例

**场景**：触摸屏读取从站01的电机高度（1250mm，存于寄存器地址0）

**请求帧（触摸屏→MCU）**：
| 01 | 03 | 00 | 00 | 00 | 01 | 84 | 0A |
|----|----|----|----|----|----|----|----|
| 地址 | 功能码 | 起始地址(0) | 数量(1) | CRC |

**响应帧（MCU→触摸屏）**：
| 01 | 03 | 02 | 04 | E2 | B9 | C8 |
|----|----|----|----|----|----|----|
| 地址 | 功能码 | 字节数(2) | 数据(0x04E2=1250) | CRC |

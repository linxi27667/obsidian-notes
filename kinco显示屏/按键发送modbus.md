# Kinco触摸屏与STM32串口通信 (Modbus RTU) 全攻略

## 1. 核心架构概述

- **通信主从：** Kinco 触摸屏作为**主站 (Master)** 发起轮询与指令，STM32 作为**从站 (Slave)** 被动响应。
    
- **通信协议：** 标准 Modbus RTU（屏幕底层自带驱动，自动打包/解包数据帧及 CRC 校验）。
    
- **物理层接口：** RS-232（全双工，通过 MAX3232 芯片进行 TTL 与 ±12V 电平转换）。
    

## 2. 硬件物理层 (RS-232连线规范)

RS-232 核心原则：**交叉接线，必须共地**。

- **STM32 端 $\rightarrow$ MAX3232 模块：**
    
    - `TX` $\rightarrow$ `TXD`
        
    - `RX` $\rightarrow$ `RXD`
        
    - `GND` $\rightarrow$ `GND`
        
    - `3.3V` $\rightarrow$ `VCC`
        
- **MAX3232 模块 $\rightarrow$ Kinco 屏幕 DB9 母头 (COM0)：**
    
    - 模块 `TX` $\rightarrow$ 屏幕 **Pin 2 (RX)**
        
    - 模块 `RX` $\rightarrow$ 屏幕 **Pin 3 (TX)**
        
    - 模块 `GND` $\rightarrow$ 屏幕 **Pin 5 (GND)**
        

## 3. 步科 (Kinco DTools) 屏幕端配置

### 3.1 添加通讯设备 (极其重要)

1. 在左侧“工程管理器”找到“通讯设置” -> `COM0`。
    
2. 右键 -> 添加设备 -> 选择 **Modbus RTU**（主站，千万别选 Slave）。
    
3. 通讯参数设置需与单片机严格对齐：**9600, 8数据位, 无校验(None), 1停止位**。站号默认为 `1`。
    

### 3.2 控件绑定与地址映射

- **不要使用** `HMIO` 和 `LW`，这是屏幕内部存储。
    
- **必须选择** 新添加的 `PLC_1` 设备，地址类型选择 **`4X` (保持寄存器)**。
    
- **按键下发指令：** 使用【多状态设定】元件，操作类型选“设常数”，数值填 `1`。按下时，屏幕会自动发送 `06` (写单个寄存器) 报文。
    
- **显示单片机数据：** 使用【数值显示】元件，屏幕会自动定时发送 `03` (读寄存器) 报文。
    

## 4. STM32 软件层开发框架 (工业级防卡死架构)

采用 **DMA + 串口空闲中断 + FreeRTOS 消息队列** 架构。

### 4.1 底层接收 (中断层)

不解析数据，只做极速搬运：

1. 开启 `HAL_UARTEx_ReceiveToIdle_DMA`。
    
2. 在空闲中断回调中，将 DMA 接收到的数据打包进 `Modbus_Msg_t` 结构体。
    
3. 使用 `xQueueSendFromISR` 发送至 FreeRTOS 队列，并立刻重启 DMA 接收。
    

### 4.2 协议解析 (任务层)

创建专用的 `Task_Modbus`，阻塞等待队列数据。收到数据后执行标准 Modbus 校验流程：

1. **长度校验：** 大于等于 8 字节。
    
2. **地址校验：** 报文第一字节必须为设定的本机地址 (`0x01`)。
    
3. **CRC 校验：** 计算前 N-2 字节的 CRC16，并与最后两字节对比（注意最后两字节是**低位在前，高位在后**）。
    
4. **动作分发：**
    
    - 收到 `03` 报文：读取 `Reg_Pool` 数组，打包成响应帧，附带 CRC 发送回去。
        
    - 收到 `06` 报文：将报文中的数值写入 `Reg_Pool` 对应地址，并将原包回传作为应答。
        

### 4.3 业务解耦

通信任务只负责维护 `Reg_Pool[50]` 这个“数据黑板”。

- **传感器任务：** 定期把温度等数据更新到 `Reg_Pool`。
    
- **执行器任务：** 不断监测 `Reg_Pool` 中按钮对应的标志位，一旦变为 `1` 就执行动作，执行完由单片机将其清 `0`。
    

## 5. 经典报文拆解示例 (屏幕按下按键)

当按下绑定了 `4X 1`、设置常数为 `1` 的按钮时，屏幕发出报文：

`[01] [06] [00 01] [00 01] [19 CA]`

- `01`：单片机设备地址。
    
- `06`：功能码（写入单个寄存器）。
    
- `00 01`：目标寄存器地址（0x0001，即 1 号柜子）。
    
- `00 01`：要写入的数值（十进制的 1）。
    
- `19 CA`：Modbus CRC16 校验码。
    

## 6. 常见踩坑与排错清单

1. **"PLC No Response" (通讯超时)：**
    
    - 90% 概率是硬件 RS232 的 TX 和 RX 没有交叉连接。
        
    - 5% 概率是单片机和屏幕没有共地。
        
    - 5% 概率是波特率等串口配置不一致。
        
2. **按键无法控制单片机：** 检查 DTools 软件中，是否选成了 `HMIO` (本机内存) 而不是 `PLC_1`。
    
3. **数据能收到但单片机不执行：** 检查 STM32 代码里的 CRC 校验高低位是否提取反了。

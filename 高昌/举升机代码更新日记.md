# 举升机代码更新日记

## 2026-04-30 W25Q128 外部 Flash 驱动开发

### 项目背景
- 平台：STM32F407VET6 + FreeRTOS (CMSIS-RTOS2)
- 通信：SPI1（硬件），CS: PA4
- Flash：W25Q128

### 文件结构

```
BSP/
├── Inc/
│   ├── bsp_spi.h          SPI 总线抽象（结构体 + 函数指针）
│   └── bsp_w25qxx.h       W25Qxx 驱动 API + 返回值宏
└── Src/
    ├── bsp_spi.c           SPI 总线操作（Init/Select/Deselect）
    └── bsp_w25qxx.c        W25Qxx 完整驱动（工业安全版）
APP/
├── Inc/
│   ├── app_spi.h           SPI1 硬件配置宏
│   └── app_w25qxx.h        W25Qxx 业务入口
└── Src/
    ├── app_spi.c           SPI 硬件绑定（hspi1 + CS 引脚）
    └── app_w25qxx.c        W25Qxx 实例化 + 系统初始化（含 SPI 初始化）
```

### 分层设计

- **BSP 层**：`spi_bus_t` 结构体 + 函数指针抽象（参考 sw_i2c 风格），零硬件头文件
- **APP 层**：唯一允许包含 HAL 头文件的地方，通过宏定义绑定硬件引脚
- 与裸机 100% 兼容，后续可直接复用

### 调用链

```
main.c → App_W25Qxx_System_Init()
  → App_SPI_System_Init()        ← 拉高 CS，初始化 SPI 总线
  → W25Q_Init_Device()           ← 等待芯片就绪 + 读 JEDEC ID 验证
    → SPI_Bus_Init()
      → HW_SPI_Init()            ← HAL_GPIO_WritePin(CS, SET)
```

### 修复的 Bug

#### Bug 1: freertos.c 编译错误
- **现象**：`ledTaskHandle` / `StartLedTask` / `ledTask_attributes` undefined
- **原因**：CubeMX 重新生成后删除了 ledTask 相关定义，但 `USER CODE BEGIN RTOS_THREADS` 里的调用残留
- **修复**：删除 freertos.c:100 的残留代码

#### Bug 2: App_SPI_System_Init 未被调用（CS 未初始化）
- **现象**：`App_W25Qxx_System_Init()` 只调了 `W25Q_Init_Device()`，没有先调 `App_SPI_System_Init()` 拉高 CS
- **原因**：`W25Q_Init_Device()` 内部不调 `SPI_Bus_Init()`，CS 引脚处于未初始化状态
- **修复**：`App_W25Qxx_System_Init()` 内部先调 `App_SPI_System_Init()`，再调 `W25Q_Init_Device()`
- **影响**：不修复会导致第一次 SPI 通信时 CS 电平不确定，可能读 ID 失败

#### Bug 3: 返回值未检查，初始化状态不可知
- **现象**：`App_W25Qxx_System_Init()` 返回 void，调用方无法判断初始化是否成功
- **修复**：新增 `App_W25Qxx_Get_JEDEC_ID()` 接口，供 main.c 查询 ID 验证通信

#### Bug 4: W25Q_Init_Device 内部读 JEDEC ID 被丢弃
- **现象**：`W25Q_Init_Device()` 内部调了 `W25Q_Read_JEDEC_ID()` 但 `(void)` 丢弃，无法通过日志确认
- **说明**：用户可在 main.c 中调用 `App_W25Qxx_Get_JEDEC_ID()` 验证（W25Q128 应返回 0xEF4018）

### 工业级安全特性（已实现）

| 特性 | 实现 |
|------|------|
| NULL 指针检查 | 所有 API 入口检查 flash/bus/buf/data |
| 跨页边界保护 | Page_Program 检测到跨页返回 W25Q_ERR |
| 扇区对齐检查 | Sector_Erase 要求 4K 对齐，否则返回错误 |
| 错误返回机制 | 所有 API 返回 W25Q_OK/W25Q_ERR |
| 自动跨页写入 | W25Q_Write_MultiPage 内部按 256 字节分页循环 |
| CS 残留保护 | 所有 SPI 事务失败时都拉高 CS 避免片选残留 |
| 线程安全 | HW_SPI_Transmit_Receive 中 dummy 为局部变量（非 static） |
| Busy 超时增大 | Wait_Busy timeout 500000，覆盖芯片级擦除 |

### W25Q128 API 速查

```c
// 初始化
App_W25Qxx_System_Init();

// 读 JEDEC ID 验证（W25Q128 返回 0xEF4018）
uint32_t id = App_W25Qxx_Get_JEDEC_ID();

// 读数据
uint8_t buf[512];
W25Q_Read_Buffer(&W25Q_Flash, 0x000000, buf, 512);

// 写数据（自动跨页）
uint8_t data[512];
W25Q_Write_MultiPage(&W25Q_Flash, 0x000000, data, 512);

// 擦除（必须 4K 对齐）
W25Q_Sector_Erase(&W25Q_Flash, 0x000000);

// 整片擦除
W25Q_Chip_Erase(&W25Q_Flash);
```

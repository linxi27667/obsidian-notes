## 三、堆垛机控制逻辑设计

### 3.1 三轴运动模型

堆垛机三轴运动：
- **X轴（行走）**：沿巷道移动，定位到目标货架列
- **Y轴（升降）**：垂直升降，定位到目标货架层
- **Z轴（伸缩）**：货叉伸缩，取放货物

| 轴 | 运动范围 | 编码器 | 电机 |
|---|---|---|---|
| X轴（行走） | 0~50m | 编码器1 | 电机1（正反转） |
| Y轴（升降） | 0~10m | 编码器2 | 电机2（正反转） |
| Z轴（伸缩） | 0~1.5m | 编码器3 | 电机3（正反转） |

### 3.4 数据结构设计

```c
// 三轴状态结构体（从举升机Column_t扩展）
typedef struct {
    int32_t  position_mm;       // 当前位置（mm）
    int32_t  target_mm;         // 目标位置（mm）
    uint32_t last_pulse_tick;   // 上次脉冲时间（堵转检测）
    uint8_t  motor_running;     // 电机运行标志
    uint8_t  direction;         // 0=停, 1=前进/上升/伸出, 2=后退/下降/收回
    uint8_t  sync_blocked;      // 同步锁定标志
    uint8_t  alarm_flag;        // 报警标志
    int32_t  pulse_count;       // 编码器脉冲计数
} Axis_t;

Axis_t axis_x = {0};  // X轴（行走）
Axis_t axis_y = {0};  // Y轴（升降）
Axis_t axis_z = {0};  // Z轴（伸缩）

// 货架位置存储结构（从举升机w25q_height_t扩展）
typedef struct {
    uint32_t magic;             // 0x53544F52 ("STOR")
    int32_t  axis_x_mm;         // X轴位置
    int32_t  axis_y_mm;         // Y轴位置
    int32_t  axis_z_mm;         // Z轴位置
    uint16_t current_column;    // 当前货架列
    uint16_t current_row;       // 当前货架层
    uint32_t crc;               // CRC32
} W25Q_Position_t;

// 系统状态枚举（从举升机扩展）
typedef enum {
    STATE_IDLE,                 // 空闲
    STATE_MOVING_X,             // X轴移动
    STATE_MOVING_Y,             // Y轴移动
    STATE_MOVING_Z,             // Z轴移动
    STATE_PICKING,              // 取货流程
    STATE_PLACING,              // 放货流程
    STATE_ALARM,                // 报警
    STATE_EMERGENCY_STOP,       // 急停
} SystemState_t;
```

### 3.5 核心控制流程

```
取货流程：
[空闲 IDLE]
    ├─ 接收目标货架位置(列,层)
    ├─ X轴移动到目标列 → [X轴移动 MOVING_X]
    ├─ Y轴升降到目标层 → [Y轴移动 MOVING_Y]
    ├─ Z轴伸出取货 → [Z轴伸出 MOVING_Z]
    ├─ Z轴收回 → [取货完成 PICKING]
    ├─ Y轴下降到出货口
    ├─ Z轴伸出放货
    └─ → [空闲 IDLE]

放货流程：
[空闲 IDLE]
    ├─ 接收目标货架位置(列,层)
    ├─ Y轴升到出货口
    ├─ Z轴伸出取货
    ├─ Z轴收回
    ├─ X轴移动到目标列 → [X轴移动]
    ├─ Y轴升降到目标层 → [Y轴移动]
    ├─ Z轴伸出放货 → [放货 PLACING]
    ├─ Z轴收回
    └─ → [空闲 IDLE]

安全保护（沿用举升机逻辑）：
[移动中]
    ├─ 限位触发 → 停止该轴
    ├─ 防撞触发 → 全局急停 + 告警
    ├─ 堵转检测(0.5s无脉冲) → 停止该轴 + 告警
    ├─ 急停按钮 → 全局急停
    └─ 松按钮 → [空闲]
```

## 四、IoT云平台设计

### 4.1 ESP32-P4网关功能

| 功能 | 实现方式 | 说明 |
|---|---|---|
| **MQTT通信** | ESP-IDF + MQTT Client | 连接阿里云IoT/OneNET |
| **WiFi连接** | ESP32内置WiFi | 支持蓝牙辅助配网 |
| **UART通信** | UART1 (115200) | 与STM32主控通信 |
| **AI异常检测** | ESP32-P4 AI加速器 | 分析运动数据，预测故障 |
| **远程控制** | MQTT订阅控制Topic | 接收云平台命令 |
| **数据上报** | MQTT发布状态Topic | 上报位置、状态、告警 |

### 4.2 MQTT Topic设计

| Topic | 方向 | 数据内容 |
|---|---|---|
| `/stacker/status` | ESP32→云平台 | 三轴位置、运行状态、告警 |
| `/stacker/command` | 云平台→ESP32 | 目标货架位置、启动/停止 |
| `/stacker/alarm` | ESP32→云平台 | 告警事件、时间、类型 |
| `/stacker/config` | 云平台→ESP32 | 参数配置（速度、阈值等） |

### 4.3 UART通信协议（STM32 ↔ ESP32）

**帧格式**：`[0xAA][长度][命令字][数据N][CRC8][0x55]`

| 命令字 | 方向 | 说明 |
|---|---|---|
| 0x01 | 双向 | 心跳包（1s一次） |
| 0x02 | STM32→ESP32 | 状态上报（三轴位置+状态+告警） |
| 0x03 | STM32→ESP32 | 告警上报 |
| 0x10 | ESP32→STM32 | 控制命令（目标位置） |
| 0x11 | ESP32→STM32 | 参数配置 |
| 0x12 | ESP32→STM32 | 启动/停止命令 |

### 4.4 云平台Dashboard设计

| 功能模块 | 说明 |
|---|---|
| **实时位置** | 三轴当前位置可视化 |
| **运行状态** | 运行/空闲/告警状态显示 |
| **告警列表** | 历史告警记录查询 |
| **远程控制** | 目标货架位置输入 + 启动按钮 |

## 六、答辩策略

### 6.1 技术问题准备

| 问题 | 回答要点 |
|---|---|
| **为什么用STM32+ESP32双芯片？** | STM32专注实时控制（FreeRTOS），ESP32专注IoT通信，职责分离，工业标准架构 |
| **堵转检测原理？** | 编码器脉冲计数，电机运行但0.5s无脉冲触发告警，比PLC的2s更快 |
| **如何保证数据掉电不丢失？** | W25Q Flash双页冗余存储，每1s自动保存，CRC校验 |
| **三轴同步如何实现？** | 比较三轴位置差值，快轴暂停等待慢轴，超时告警 |
| **AI异常检测做什么？** | ESP32-P4分析运动加速度数据，预测轴承磨损、电机异常 |

## 十、电机选型与控制方案（详细）

### 10.1 三种控制方案对比

| 方案 | 控制方式 | 编码器需求 | 精度 | 成本 | 推荐度 |
|---|---|---|---|---|---|
| **方案A：继电器+编码器闭环** | 继电器控制交流电机 + 编码器反馈 | ✅ 必须 | ±5mm | 中 | **与现有代码兼容** |
| **方案B：步进电机开环** | STM32发脉冲→步进电机 | ❌ 不需要 | ±0.05mm | 低 | **大赛演示首选** |
| **方案C：变频器+编码器闭环** | PWM/模拟量→变频器 + 编码器反馈 | ✅ 必须 | ±2mm | 中高 | **工业级方案** |

### 10.2 方案A：继电器+编码器闭环（与现有代码兼容）

**原理**：你的举升机代码就是这种方案
- STM32输出GPIO→继电器→交流电机正反转
- 编码器脉冲→STM32 TIM输入捕获→位置反馈
- 控制逻辑：达到目标位置→停止电机

**优点**：
- ✅ 代码改动量最小（直接用现有代码）
- ✅ 适合工业级答辩（真实堆垛机也是这样）

**缺点**：
- ❌ 需要编码器硬件
- ❌ 精度较低（±5mm）
- ❌ 速度不可调（继电器只有开/关）

**需要的硬件**：
| 轴 | 电机 | 编码器 | 继电器/接触器 | 成本 |
|---|---|---|---|---|
| X轴 | 交流异步电机 0.75kW | 旋转编码器 1000脉冲/转 | 正反转接触器2个 | ~500元 |
| Y轴 | 交流异步电机 1.5kW + 电磁刹车 | 旋转编码器 1000脉冲/转 | 正反转接触器2个 | ~700元 |
| Z轴 | 交流异步电机 0.4kW | 旋转编码器 500脉冲/转 | 正反转接触器2个 | ~300元 |
| **合计** | - | - | - | **~1500元** |

---

### 10.3 方案B：步进电机开环控制（大赛演示首选）

**原理**：步进电机自带位置控制
- STM32输出脉冲信号→步进驱动器→步进电机
- 每个脉冲 = 电机转一个固定角度（如1.8°/脉冲）
- 发N个脉冲 = 电机转N步 = 精确位置
- **不需要编码器反馈！**

**优点**：
- ✅ 不需要编码器，成本最低
- ✅ 精度最高（±0.05mm）
- ✅ 速度可调（改变脉冲频率）
- ✅ 控制代码简单（只需发脉冲）

**缺点**：
- ❌ 需要改代码（从继电器控制改为PWM脉冲控制）
- ❌ 大功率场景不适合（步进电机功率有限）
- ❌ 可能丢步（过载时位置不准）

**需要的硬件**：
| 轴 | 步进电机 | 驱动器 | STM32接口 | 成本 |
|---|---|---|---|---|
| X轴 | 57步进电机 2N·m | TB6600驱动器 | 2路GPIO(PULSE/DIR) | ~200元 |
| Y轴 | 86步进电机 4N·m + 抱闸 | TB6600驱动器 | 2路GPIO + 抱闸控制 | ~350元 |
| Z轴 | 42步进电机 1N·m | TB6600驱动器 | 2路GPIO | ~150元 |
| **合计** | - | - | - | **~700元** |

**步进电机控制原理图**：
```
STM32 GPIO ────┬──→ PULSE（脉冲）──→ 驱动器 ──→ 步进电机转1步
               │
               └──→ DIR（方向）─────→ 正转/反转
               
目标位置计算：
  目标脉冲数 = 目标距离(mm) / 每脉冲行程(mm)
  
示例：丝杆导程5mm/转，步进电机200步/转
  每脉冲行程 = 5mm / 200 = 0.025mm
  移动100mm需要脉冲 = 100 / 0.025 = 4000个脉冲
```

**STM32控制代码（步进电机）**：
```c
// 步进电机控制（替代继电器控制）
void Stepper_Move(axis_t axis, int32_t target_mm) {
    // 计算目标脉冲数
    int32_t pulses = target_mm / PULSE_PER_MM;
    
    // 设置方向
    GPIO_Write(DIR_PORT[axis], target_mm > current_pos ? 1 : 0);
    
    // 发脉冲（PWM或GPIO翻转）
    for (int i = 0; i < pulses; i++) {
        GPIO_Write(PULSE_PORT[axis], 1);
        delay_us(10);  // 脉冲宽度
        GPIO_Write(PULSE_PORT[axis], 0);
        delay_us(PULSE_INTERVAL);  // 脉冲间隔（控制速度）
    }
    
    // 更新当前位置（软件计数）
    current_pos[axis] = target_mm;
}

// 更精确的方式：用TIM PWM输出脉冲
void Stepper_PWM_Start(axis_t axis, int32_t speed_hz) {
    TIM_SetFrequency(htim[axis], speed_hz);  // 设置PWM频率=速度
    TIM_PWM_Start(htim[axis]);  // 开始输出脉冲
    // 计数器自动计数脉冲数，达到目标后停止
}
```

### 10.4 方案C：变频器+编码器闭环（工业级方案）

**原理**：变频器控制交流电机速度
- STM32输出PWM/模拟量（0-10V）→变频器→电机速度控制
- 编码器反馈→STM32→位置闭环
- 达到目标位置→变频器减速停止

**优点**：
- ✅ 工业标准方案（真实堆垛机用这个）
- ✅ 速度可调、启停平滑
- ✅ 适合大功率电机

**缺点**：
- ❌ 成本高（变频器贵）
- ❌ 需要编码器
- ❌ 代码改动量大

**需要的硬件**：
| 轴 | 电机 | 变频器 | 编码器 | 成本 |
|---|---|---|---|---|
| X轴 | 交流电机 0.75kW | 变频器 1kW | 旋转编码器 | ~800元 |
| Y轴 | 交流电机 1.5kW | 变频器 2kW | 旋转编码器 | ~1000元 |
| Z轴 | 交流电机 0.4kW | 变频器 0.5kW | 旋转编码器 | ~500元 |
| **合计** | - | - | - | **~2300元** |

| 你的需求 | 推荐方案 | 原因 |

**最终推荐**：**方案B（步进电机开环）**

- 精度最高

```cpp
const char* mqtt_server = "192.168.1.100";  // 你的服务器IP
const int mqtt_port = 1883;

WiFiClient espClient;
PubSubClient client(espClient);

void setup() {

    client.setServer(mqtt_server, mqtt_port);
    
    if (client.connect("esp32-stacker")) {

        client.publish("stacker/status", "ESP32已连接");
    }
}

void loop() {
    client.loop();
    
    // 定期上报状态
    static unsigned long lastPub = 0;
    if (millis() - lastPub > 5000) {
        char msg[100];
        sprintf(msg, "{\"x\":%d,\"y\":%d,\"z\":%d}", x_pos, y_pos, z_pos);
        client.publish("stacker/status", msg);
        lastPub = millis();
    }
}
```

## 十四、答辩常见技术问题回答

### Q1：为什么用步进电机不用伺服？

> **回答**：工业堆垛机90%以上实际使用变频器控制的普通异步电机，伺服电机成本高3-5倍，只在高速精密场景才需要。步进电机精度±0.05mm远超堆垛机±5mm的要求，成本可控，适合大赛演示。

### Q2：步进电机开环会不会丢步？

> **回答**：步进电机在额定负载下不会丢步。我们设计时选择2倍额定扭矩的安全余量，并设置了限位开关双重保护。即使丢步，限位开关也能触发安全停止。

### Q3：AI异常检测具体做什么？

> **回答**：我们用Isolation Forest算法分析电机运行数据（电流、振动），可以预测轴承磨损、电机异常。简化版本用阈值检测：电流超过正常值150%持续500ms触发堵转预警。

### Q4：为什么不用阿里云IoT商业平台？

> **回答**：私有MQTT服务器（Mosquitto）更灵活，数据不外流，适合工业场景。Mosquitto是开源免费，部署简单，性能足够支持百级设备连接。

### Q5：三轴同步如何实现？

> **回答**：我们采用"快轴等待慢轴"策略：实时比较三轴位置差值，如果某轴偏差超过阈值，暂停快轴等待慢轴跟上。这与举升机的双柱同步算法原理相同。
